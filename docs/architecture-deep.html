MODULE 01

Agent Loop — ReAct 상태머신

dbcode의 심장. 사용자 입력을 받아 LLM 호출 → 도구 실행 → 결과 피드백을 반복하는 핵심 루프입니다.

  • 새 도구 추가 시: ToolRegistry에 등록만 하면 자동으로 Agent Loop가 인식. 루프 코드 수정 불필요

  • Dual Model 확장: 현재 plan/execute/review 3단계 → 향후 debug, test 등 추가 가능

Hot Tools vs Deferred Tools

🔥 Hot Tools (6개) — 항상 로딩

매 LLM 요청마다 스키마가 포함되는 핵심 도구. 빠른 응답을 위해 Eager Loading.

file_read file_write file_edit bash_exec glob_search grep_search

💤 Deferred Tools (MCP) — 필요 시 로딩

MCP 서버의 도구들. 시스템 프롬프트에 이름만 노출하고, LLM이 요청하면 스키마를 로딩.

mcp__playwright__click
mcp__serena__find_symbol
mcp__chrome-devtools__navigate_page
... (수백 개 가능)

🧭 구현 방향 & 확장 포인트

  • 새 도구 추가: src/tools/definitions/에 파일 생성 → ToolDefinition 구현 → registry에 registerAll()
  • Adaptive Schema 커스텀: 모델별 few-shot 예시를 adaptive-schema.ts에서 조정
  • Tool Grouping 최적화: 현재 읽기/쓰기 분류 → 파일 경로 충돌 분석으로 세분화 가능
  • 도구 메트릭 수집: 실행 시간, 성공률, 토큰 비용 등 수집하여 도구 추천에 활용

5가지 권한 모드

default

매번 사용자에게 확인을 요청합니다. 가장 안전한 모드.

acceptEdits

파일 읽기/쓰기/수정을 자동 승인. bash 명령은 여전히 확인 필요.

plan

읽기 전용 도구만 자동 승인. 쓰기 작업은 모두 확인 필요.

dontAsk

도구당 1회만 확인 후 세션 동안 기억합니다.

bypassPermissions

모든 도구를 무조건 허용. ⚠️ 개발/테스트 전용, 프로덕션에서 사용 금지.

🎯
규칙 패턴 매칭: ToolName(arg1: value1, arg2: *) 형식으로 glob 패턴 지원. 예: bash_exec(command: npm *)는 npm으로 시작하는 모든 명령을 매칭합니다. ParsedPermissionPattern으로 파싱 후 빠른 매칭.

🧭 구현 방향 & 확장 포인트

  • 감사 로그 활용: audit-log.ts의 JSONL 데이터로 도구 사용 패턴 분석 대시보드 구현 가능
  • 팀 권한 정책: 프로젝트 .dbcode/settings.json으로 팀 공통 deny/allow 규칙 공유
  • 동적 권한 학습: 사용자 승인 패턴 분석 → 자동 규칙 제안 기능

핵심 아키텍처 결정 10가지

단방향 레이어 의존성

CLI → Core → Infra → Leaf. 절대 역방향 import 금지. madge --circular로 검증.

AbortSignal 패턴

모든 장시간 작업은 AbortController로 취소 가능. Esc 키 → 즉시 중단.

Immutable State

readonly 프로퍼티 + spread copy. 상태 변경 추적이 쉽고 버그 발생률 감소.

3-Layer 토큰 관리

Microcompaction → Auto-compaction → Rehydration으로 컨텍스트 한계를 효율적으로 관리.

Dual Model 패턴

Architect(추론) vs Editor(실행) 분리로 비용 최적화 + 품질 향상 동시 달성.

Deferred Tool Loading

MCP 도구 스키마를 On-demand 로딩. 수백 개 도구에도 토큰 소모 최소화.

Deny-First 권한

deny 규칙 최우선. 보안 사고 방지를 기본 원칙으로 설계.

에러 유형별 복구

에러를 분류하고 각 유형에 최적화된 전략을 자동 실행.

Circuit Breaker

무한 루프 방지. 무변경/에러 반복 감지 → 자동 중단 + 복구 시도.

Graceful Degradation

잘못된 설정이나 패턴은 조용히 건너뜀. 하나의 문제가 전체를 멈추지 않도록.
