### 3.2 Agent Loop 반복별 누적 토큰

dbcode Agent Loop에서 **입력 토큰은 반복마다 누적**됩니다:

```
반복 i의 Input = SystemPrompt + Σ(j=1→i-1)[AssistantOutput_j + ToolResults_j]
```

| 반복(Iteration) | 누적 Input | 신규 Output | 누적 합계 | 비고                  |
| :-------------: | ---------: | ----------: | --------: | --------------------- |
|        1        |      7,000 |       1,200 |     8,200 | System + 첫 요청      |
|        2        |      9,500 |       1,500 |    11,000 | +파일 읽기 결과       |
|        3        |     12,500 |       2,000 |    14,500 | +코드 생성            |
|        5        |     20,000 |       1,500 |    21,500 | 누적 가속             |
|        8        |     32,000 |       1,200 |    33,200 |                       |
|       10        |     42,000 |       1,000 |    43,000 |                       |
|       15        |     68,000 |         800 |    68,800 | 압축 임계 근접        |
|       20        |     95,000 |       1,000 |    96,000 | **83.5% 압축 트리거** |

> **컨텍스트 압축 후**: 95K → ~45K로 감소 (50% 절감), 추가 LLM 요약 호출 1회 발생

### 3.3 세션 단위 토큰 공식

하나의 리팩토링 세션(일관된 작업 단위)의 토큰 비용:

```
Session Input  = N × S + T × N(N+1)/2    (압축 전)
Session Output = N × O_avg

여기서:
  N = 반복 횟수
  S = System Prompt (DBCODE.md 포함 ~7,000 토큰)
  T = 반복당 추가 컨텍스트 (~2,000 토큰)
  O_avg = 반복당 평균 출력 (~1,200 토큰)
```

| 세션 규모          | 반복 | Input 토큰 | Output 토큰 | 세션 합계 |
| ------------------ | :--: | ---------: | ----------: | --------: |
| 소형 (2-3파일)     |  6   |     66,000 |       7,200 |    73,200 |
| 중형 (5-8파일)     |  10  |    180,000 |      12,000 |   192,000 |
| 대형 (10+파일)     |  15  |    330,000 |      18,000 |   348,000 |
| 초대형 (모듈 전체) | 20+  |   460,000+ |     24,000+ |  484,000+ |

> 컨텍스트 압축이 발생하면 후반부 반복의 input이 감소하므로, 대형 세션은 이론치보다 **20-30% 적게** 소비됩니다.
