## 3. Agent 실행 패턴 분석

### 3.1 Agent Loop 동작 방식

dbcode는 **ReAct (Reasoning + Acting) 패턴**으로 동작합니다:

```
User Input → System Prompt → LLM 호출 → Tool 추출 → 권한 확인
→ 체크포인트 → Tool 실행 → 결과 추가 → 다음 반복 or 완료
```

| 설정                 | 값       | 의미                    |
| -------------------- | -------- | ----------------------- |
| maxIterations        | 50       | 무한루프 방지           |
| maxRetries           | 2        | LLM 호출 실패 시 재시도 |
| 컨텍스트 압축 임계값 | 83.5%    | 자동 요약 트리거        |
| Tool 결과 최대 길이  | 12,000자 | 결과 잘림 방지          |
| 응답 예약 비율       | 20%      | 출력 공간 확보          |

### 3.2 Subagent 시스템

| Agent 타입    | Max 반복 | 도구 접근 | 용도            |
| ------------- | :------: | --------- | --------------- |
| explore       |    15    | 읽기 전용 | 코드베이스 탐색 |
| plan          |    10    | 읽기 전용 | 아키텍처 설계   |
| general       |    25    | 전체 14개 | 범용 작업       |
| code-reviewer |    15    | 읽기 전용 | 코드 리뷰       |
| test-runner   |    20    | 읽기+bash | 테스트 실행     |

### 3.3 병렬 Tool 실행 최적화

dbcode는 읽기 전용 도구를 자동으로 병렬 실행합니다:

```
GROUP 1 (병렬): glob_search + file_read + grep_search
GROUP 2 (병렬): file_write(A.tsx) + file_write(B.tsx)
GROUP 3 (순차): bash_exec npm install
```

7개 도구 호출 → **3개 그룹** = 3번의 LLM 반복만 필요 (60% 절약)

## 4. 반복(Iteration)별 토큰 소비 모델

### 4.1 단일 반복 토큰 구조

| 구성 요소               |             토큰 수 | 비고                   |
| ----------------------- | ------------------: | ---------------------- |
| System Prompt           |       3,500 ~ 5,500 | DBCODE.md 포함 시 증가 |
| 대화 히스토리 (누적)    | 반복마다 +500~2,000 | 이전 턴 누적           |
| User Message            |           100 ~ 500 | 초기 프롬프트          |
| Tool 결과 (누적)        | 반복마다 +200~2,000 | 파일 내용 등           |
| **Input 합계 (1회차)**  |          **~5,000** |                        |
| **Input 합계 (5회차)**  |         **~12,000** | 누적 효과              |
| **Input 합계 (10회차)** |         **~25,000** |                        |
| **Input 합계 (20회차)** |        **~50,000+** | 압축 트리거 가능       |
| Assistant Output (1회)  |         300 ~ 1,500 | 코드 생성 시 증가      |

### 4.2 누적 토큰 공식 (컨텍스트 압축 전)

```
Total Input Tokens ≈ Σ(i=1→N) [SystemPrompt + i × AvgTurnSize]
                   ≈ N × SystemPrompt + N(N+1)/2 × AvgTurnSize

Total Output Tokens ≈ N × AvgOutputPerTurn
```
