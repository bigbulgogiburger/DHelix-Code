핵심 문제점 — 왜 "낫지 않다"는 평가인가

1. 코드 수정 후 자동 품질 검증 루프 부재

코드 수정 후 lint/typecheck/test를 자동으로 돌려주는 피드백 루프가 없음. 참고: 네이티브 LSP는 OpenCode만 보유하고 Claude Code/Codex/Gemini CLI도 LSP 없음. 하지만 Claude Code는 bash로 빌드 도구 실행, Aider는 auto lint/test 루프를 돌림. dhelix는 두 방식 모두 없어 사용자가 직접 빌드해봐야 함.

2. 세션 영속성 없음

OpenCode는 Go 기반 persistent server로 터미널 끊김, SSH 드롭에도 세션 유지. dhelix는 프로세스 종료 = 세션 소멸. 장시간 작업에 치명적.

3. 선언형 에이전트 커스터마이즈 부재

OpenCode는 YAML frontmatter로 커스텀 에이전트를 선언적 정의. dhelix는 코드 수정 필요. 파워유저의 확장성 진입장벽이 높음.

4. 데스크톱 앱 & 멀티플랫폼 부재

OpenCode는 Tauri 기반 데스크톱, Claude Code는 IDE 통합. dhelix는 순수 터미널만. 비-CLI 사용자에게 진입장벽.

5. 벤치마크 / 가시적 성과 부재

Claude Code는 SWE-bench 80.9%, Codex는 Terminal-Bench 77.3%. dhelix는 벤치마크 결과 없음. 설득력 부족.

💡 핵심 인사이트: dhelix의 내부 아키텍처(ReAct Loop, 3-Layer Compaction, 5-Mode Permission)는 실제로 잘 설계되어 있다. 문제는 "보이지 않는 곳에 투자"했다는 것. 사용자가 체감하는 DX(개발자 경험)와 가시적 차별화 기능에서 OpenCode에 뒤처진다.

03 — Gap Analysis

Critical Gap 분석

dhelix가 반드시 메워야 할 격차와 역전 가능한 기회

💪 Strengths (dhelix 우위)

  • 3-Layer Context Compaction — 대부분 경쟁자는 단순 truncation. Cold Storage + Rehydration은 고유 강점
  • 5-Layer Security Guardrails — Command/Path/Secret/Injection/Entropy 다층 방어. 엔터프라이즈급 보안
  • 계층형 권한 시스템 — 패턴 기반 규칙(Bash(npm *)), 감사 로그. OpenCode보다 세밀
  • Dual-Model Router — Architect/Editor 패턴. 비용 최적화 내장
  • 풍부한 CLI 명령어 — 41개 슬래시 명령어. 파워 유저 친화적
  • Extended Thinking 지원 — Claude 모델의 thinking budget 활용
  • 음성 입력 — CLI에서 음성 지원하는 유일한 에이전트

⚠️ Weaknesses (dhelix 열위)

  • 자동 품질 검증 루프 없음 — 파일 수정 후 lint/typecheck 자동 실행 없음. 네이티브 LSP는 OpenCode만 보유(Claude Code/Codex도 없음), 하지만 Aider의 auto lint/test 루프조차 없어 체감 격차 큼
  • Repo Map / 코드 인덱싱 부재 — Aider의 Tree-sitter 기반 repo map 대비 원시적
  • 세션 영속성 취약 — 파일 기반 복원만. daemon/SQLite 수준에 못 미침
  • 선언형 에이전트 부재 — 커스텀 에이전트 정의에 코드 수정 필요
  • 벤치마크 부재 — SWE-bench, Terminal-Bench 결과 없음
  • 느린 Go 대비 부트스트랩 — Node.js 런타임 오버헤드 (2-3초 vs Go의 <500ms)
  • 커뮤니티 / 생태계 부재 — 오픈소스 기여자, 플러그인 마켓 없음
  • 자동 lint/test 루프 없음 — Aider는 코드 생성 후 자동 lint/test/fix

🚀 Opportunities (기회)

  • Context Flywheel — 조직 컨텍스트 축적이 진짜 해자(moat). dhelix의 Memory 시스템을 강화하면 경쟁 우위
  • 엔터프라이즈 보안 니즈 — 규제 환경에서 로컬 우선 + 5-Layer 가드레일은 큰 장점
  • 비용 효율성 경쟁 — "4x 적은 토큰으로 같은 결과"가 핫 토픽. Compaction + Dual Model 강화
  • 한국 시장 특화 — 한국어 UX, 국내 클라우드(NHN, Naver) 통합, 한국 개발 문화 반영
  • AI Agent 표준화 물결 — ACP(Agent Communication Protocol) 등 새로운 표준 선점
  • IDE 통합 대세 — Zed, JetBrains, VS Code 플러그인으로 확장

🔥 Threats (위협)

  • Big Tech 자원 격차 — Anthropic, Google, OpenAI가 공식 CLI 에이전트 투입
  • 오픈소스 네트워크 효과 — OpenCode 112K⭐, Aider 30K⭐. 커뮤니티 기여가 격차 확대
  • 모델 종속성 — Claude/GPT API 가격 정책 변동에 취약
  • AI 코딩 도구 범용화 — 에이전트 기능이 코모디티화. 차별화 어려움
  • IDE 에이전트 확대 — Cursor, Windsurf가 터미널 에이전트 기능 흡수
04 — Strategic Direction

전략적 방향: 3가지 핵심 전략

"모든 것을 따라하지 말고, 이길 수 있는 싸움을 골라라"

dhelix의 Positioning

"가장 지능적인 코드 이해"와 "가장 안전한 엔터프라이즈 에이전트"의 교차점.
범용 에이전트가 아닌, "코드를 가장 깊이 이해하는 Secure-First Coding Agent"로 포지셔닝.

🧠 전략 1: Code Intelligence Layer — "코드를 이해하는 에이전트"

CLI 코딩 에이전트 중 네이티브 LSP 내장은 OpenCode만 유일하다. Claude Code, Codex, Gemini CLI 모두 LSP가 없다. 그러나 Claude Code는 bash로 빌드 도구를 실행하고, Aider는 자동 lint/test 루프를 돈다. dhelix는 이 두 접근 모두 없어 체감 격차가 크다. 현실적 단계별 전략을 취한다.

🔄 Auto Lint/Test/Typecheck Loop

코드 수정 후 자동 lint → typecheck → test 실행. 실패 시 에러를 LLM에 피드백 → 자동 수정. 기존 Hook + bash_exec 활용으로 2주 내 MVP 가능. Aider/Claude Code 접근.

CRITICAL — Phase 1 최우선

🗺 Semantic Repo Map

Tree-sitter 기반 AST 파싱 → 함수/클래스/모듈 관계도 자동 생성. Aider의 repo-map을 벤치마킹하여, 의존성 그래프 + 호출 체인 제공.

CRITICAL

🔗 LSP Integration Hub

TS(tsserver), Go(gopls), Python(pylsp) LSP 자동 감지 & 연결. OpenCode만 가진 기능을 흡수하여 차별화. Auto Loop 안정화 후 진행.

HIGH — Phase 2 차별화

🛡 전략 2: Enterprise-Grade Trust — "가장 신뢰할 수 있는 에이전트"

이미 보유한 5-Layer Guardrails + Permission System을 엔터프라이즈 킬러 피처로 발전시킨다. 규제 산업(금융, 의료, 공공)에서 "유일하게 쓸 수 있는 코딩 에이전트"가 되는 것이 목표.

🔐 Policy-as-Code Engine

OPA(Open Policy Agent) 스타일 정책 엔진. 팀별/프로젝트별 보안 정책을 코드로 관리. 기존 권한 시스템을 기업용으로 확장.

HIGH

📊 Compliance Dashboard

에이전트의 모든 행동을 추적하는 감사 대시보드. "어떤 파일을 수정했는지, 어떤 명령을 실행했는지" 완전한 추적성.

MEDIUM

🏢 Air-Gap Mode

완전 오프라인 모드. Ollama/vLLM + 로컬 MCP 서버만으로 동작. 군사/보안 기관, 금융권에서의 유일한 선택지.

HIGH

⚡ 전략 3: Context Flywheel — "쓸수록 똑똑해지는 에이전트"

2026년의 핵심 경쟁 우위는 "조직의 도메인 컨텍스트를 얼마나 깊이 축적하는가". dhelix의 Memory 시스템을 Context Flywheel로 발전시켜, 사용할수록 더 정확해지는 에이전트를 만든다.

🧬 Project DNA

프로젝트의 코딩 컨벤션, 아키텍처 패턴, 자주 쓰는 라이브러리를 자동 학습. "이 프로젝트에서는 이렇게 코딩한다"를 아는 에이전트.

CRITICAL

📈 Decision Memory

과거 결정의 이유(WHY)를 기록. "이전에 Redis 대신 Memcached를 선택한 이유"를 기억하는 에이전트. 기존 memory 시스템을 structured decisions로 확장.

HIGH

🔄 Team Knowledge Sync

팀원들의 에이전트 경험을 공유 메모리로 통합. "팀 전체가 축적한 컨텍스트"를 활용하는 진정한 팀 에이전트.

MEDIUM
💡 전략적 차별화 원칙: OpenCode는 "유연성", Claude Code는 "지능", Aider는 "경량". dhelix는 "Code Intelligence + Enterprise Trust + Context Flywheel"의 삼각형으로 독자적 포지션을 구축. 이 세 축은 서로 시너지를 내며, 특히 엔터프라이즈 시장에서 압도적 우위를 만든다.
