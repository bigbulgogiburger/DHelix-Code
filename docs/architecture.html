CORE ENGINE

Agent Loop (ReAct Pattern)

dbcode의 심장부. 사용자 입력을 받아 LLM과 도구를 오케스트레이션합니다.

Agent Loop 실행 흐름

각 단계별 설명

Context Prepare

사용자 메시지를 컨텍스트에 추가하고, 시스템 프롬프트를 조립합니다. 컨텍스트 사용률이 83.5%를 넘으면 자동 압축이 트리거됩니다.

Input Filter (Guardrails)

보안 필터를 통과시킵니다: 프롬프트 인젝션 감지, 비밀 키 스캔, 위험 명령 차단 등.

LLM Stream

LLM에 요청을 보내고 SSE(Server-Sent Events)로 응답을 스트리밍합니다. 백프레셔 관리 포함.

Tool Grouping & Parallel Execute

LLM이 요청한 도구 호출을 충돌 분석합니다. 같은 파일 쓰기는 순차 실행, 읽기 전용 도구는 병렬 실행합니다.

Observation Masking

재생성 가능한 도구 출력(file_read, grep 등)을 마스킹하여 컨텍스트 윈도우를 절약합니다.

Circuit Breaker

3회 연속 무변경 또는 5회 같은 에러 반복 시 트립됩니다. Recovery Executor로 위임하여 compact/retry/fallback 전략을 실행합니다.

에러 복구 전략

에러 패턴전략설명재시도
request too largecompact대화 압축 후 재시도1회
ETIMEDOUT / timeoutretry지수 백오프로 재시도2회
parse error / invalid JSONfallback텍스트 파싱 전략으로 전환1회
ELOCK / lockedretry1초 백오프로 재시도3회

TOOL SYSTEM

16 Built-in Tools

AI가 코딩 작업을 수행하기 위해 사용하는 도구들. 각 도구는 권한 레벨이 지정되어 있습니다.

🔵
Hot Tools (파란 점 표시)는 항상 LLM 스키마에 포함되는 핵심 도구입니다. 나머지는 필요할 때 Lazy Loading됩니다.

파일 읽기. 줄 번호, offset/limit, 이미지/PDF 지원. 안전하므로 자동 실행됩니다.

도구 호출 전략 (Tool-Call Strategy)

Native Function Calling

GPT-4o, Claude 등 함수 호출을 네이티브로 지원하는 모델용. 가장 정확하고 효율적입니다.

Text Parsing

XML 태그 기반 폴백. 함수 호출을 지원하지 않는 모델에서 사용합니다.

Two-Stage

복잡한 워크플로우용. 먼저 계획을 세우고, 그 다음 실행하는 2단계 접근.

Adaptive Schema (모델 능력 기반)

능력 티어스키마 상세도로딩 방식설명
HIGHFullEager모든 파라미터, 설명, 예시 포함
MEDIUMReducedLazy필수 파라미터만, 최소 설명
LOWMinimalOn-demand이름 + 필수 파라미터 + few-shot 예시

MCPManager.connectAll()

MCPClient.connect()

MCPToolBridge.registerTools()

mcp__{server}__{tool} 네이밍
