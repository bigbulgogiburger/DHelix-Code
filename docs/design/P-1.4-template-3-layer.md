# P-1.4 — Template 3-Layer Hierarchy 설계서

**작성일**: 2026-04-23
**상태**: **v0.2 — Open Questions 해결 + Exit Criteria 충족** (Phase -1 설계, carryover)
**관련**: PRD §7.2 신규 모듈 레이아웃 (`.dhelix/templates/{primitives,patterns,project}/`) / archive/hardening §4 E4
**구현 시점**: Phase 2 Week 2 (Artifact generators)

**이 문서의 역할**: Template의 3계층 (primitives / patterns / project)의 해석 우선순위, 파일 구조, include 규칙을 확정.

---

## 0. TL;DR

- **primitives**: dhelix release에 내장된 최소 템플릿 (hook, skill, agent, command 기본 골격)
- **patterns**: dhelix가 배포하는 추천 패턴 (OWASP gate, DDD review 등). 사용자가 선택/수정 가능
- **project**: 사용자가 프로젝트에 추가한 custom 템플릿
- 해석 우선순위: **project > patterns > primitives** (같은 templateId면 project가 이김)
- 템플릿 엔진: Handlebars (`.hbs` 확장자)

## 2. Template 해석 우선순위

```typescript
// src/recombination/generators/template-resolver.ts
export type TemplateLayer = "project" | "patterns" | "primitives";

export interface ResolvedTemplate {
  readonly templateId: string;
  readonly layer: TemplateLayer;
  readonly path: string;
  readonly content: string;
}

export async function resolveTemplate(templateId: string): Promise<ResolvedTemplate> {
  // 1) project/<templateId>.hbs (프로젝트 custom)
  const projectPath = `.dhelix/templates/project/${templateId}.hbs`;
  if (await exists(projectPath)) return { templateId, layer: "project", path: projectPath, content: await readFile(projectPath, "utf-8") };

  // 2) patterns/<templateId>.hbs (dhelix 배포)
  const patternsPath = `${HOME}/.dhelix/templates/patterns/${templateId}.hbs`;
  if (await exists(patternsPath)) return { templateId, layer: "patterns", path: patternsPath, content: await readFile(patternsPath, "utf-8") };

  // 3) primitives/<kind>.hbs (최후)
  const primitivesPath = `${HOME}/.dhelix/templates/primitives/${templateId}.hbs`;
  if (await exists(primitivesPath)) return { templateId, layer: "primitives", path: primitivesPath, content: await readFile(primitivesPath, "utf-8") };

  throw new Error(`Template not found: ${templateId}`);
}
```

**주의**: `templateId`는 파일명 stem (확장자 제외). `hook` vs `owasp-gate` vs `ddd-review` 등. 충돌 방지를 위해 kebab-case.

## 3. Handlebars 변수 계약

Plasmid interpreter가 반환한 `artifacts` 배열의 `vars`를 Handlebars context로 주입:

```yaml
artifacts:
  - kind: hook
    event: PreToolUse
    matcher: { tool: Bash, commandPattern: "^git commit" }
    templateId: owasp-gate
    vars:
      severityThreshold: high
      categoriesPriority: [A01, A03, A07]
```

Handlebars 템플릿 (`.dhelix/templates/patterns/owasp-gate.hbs`):

```handlebars
// Generated by plasmid: {{plasmid.name}}
// Severity threshold: {{vars.severityThreshold}}

import { spawnSync } from "child_process";

export default async function preToolUseHook(ctx) {
  if (ctx.tool !== "Bash") return;
  if (!/{{matcher.commandPattern}}/.test(ctx.command)) return;

  const priorities = [{{#each vars.categoriesPriority}}"{{this}}"{{#unless @last}},{{/unless}}{{/each}}];
  const result = spawnSync("owasp-scan", ["--threshold", "{{vars.severityThreshold}}"]);
  if (result.status !== 0) {
    return { block: true, reason: "OWASP scan failed" };
  }
}
```

### 3.1 제공되는 변수

| Variable | 타입 | 출처 |
|----------|-----|-----|
| `plasmid.name` | string | plasmid frontmatter |
| `plasmid.description` | string | |
| `plasmid.priority` | string | |
| `plasmid.tier` | string | |
| `matcher` | object | artifacts[].matcher (hook인 경우) |
| `event` | string | hook event name |
| `vars` | object | artifacts[].vars (사용자 + LLM 합성) |
| `project.name` | string | package.json name |
| `project.language` | string | P-1.8 detect |

## 5. Primitives vs Patterns의 경계

**Primitives** (6개 고정):
- 절대 변경 안 됨 (dhelix release 포함)
- 가장 방어적 default (trustLevel safe, allowedTools 빈 배열, triggers 빈 배열)
- Generator가 "slot filling 실패" 시 폴백으로 사용

**Patterns** (n개, 증가):
- dhelix 팀이 추천하는 실용 패턴
- 사용자가 `--template <id>` 로 명시 선택 가능
- 새 패턴 추가는 semver minor bump (+ release notes)

**Project** (사용자 custom):
- 프로젝트 특유 제약 반영 (ex: 내부 CI 도구 통합)
- 버전 관리 대상 (git 포함 권장)

```

    templateId = artifact.templateId ?? defaultTemplateForKind(artifact.kind)
```

## 9. Resolved Questions (v0.2)

### Q1 — Primitives vs patterns 네이밍 충돌 방지

**확정**: **네이밍 컨벤션 강제 + 로드 시점 검증**.

- Primitives: generic one-word (`hook`, `skill`, `agent`, `command`, `harness`, `prompt-section`) — 6개 고정
- Patterns: specific kebab-case 2+ words (`owasp-gate`, `ddd-review`, `otel-observer`) — 최소 2 hyphen segments
- `template-resolver.ts` 로드 시 pattern 네임 검증: `/^[a-z0-9]+(-[a-z0-9]+)+$/` (최소 1 하이픈)
- 위반 시 load error: `TEMPLATE_NAMING_VIOLATION`

**Rationale**: 네임스페이스 분리로 충돌 불가능 (primitive는 단어 1개, pattern은 ≥2). 규약 강제로 future-proof.

### Q2 — Handlebars helper 범위

**확정**: **5개 core helper 채택, Phase 4+ 확장 검토**.

```typescript
// src/recombination/generators/handlebars-helpers.ts
export const HELPERS = {
  camelCase: (s: string) => ...,
  titleCase: (s: string) => ...,
  kebabCase: (s: string) => ...,
  ifIncludes: (arr: unknown[], val: unknown, opts) => ...,
  join: (arr: string[], sep: string) => arr.join(sep),
};
```

추가 요청 발생 시 Phase 4+ 에서 lodash-ish 확장 (`pluralize`, `truncate`, `default`).

**Rationale**: Minimal surface area → template이 logic 덩어리 되는 것 방지 (template 철학: view only).

### Q3 — Project layer 공유 메커니즘

**확정**: **v0.4 git only**. `~/.dhelix/templates/team/` 은 v0.5+ 로 연기.

**Rationale**:
- Git 통한 공유는 이미 팀에 구축된 pipeline (PR review, branch protection) 활용
- 별도 team registry 는 governance 복잡 (서명 검증, 업데이트 알림 등) → v0.5 이후 의미

**안내 문서**: Phase 6 Polish 시점에 `docs/team-templates.md` 작성 (git workflow 권장).

### Q4 — Template 업데이트 시 artifact 재생성

**확정**: **plasmid `.compiled.lock.json` 에 `templateHashes: Record<templateId, hash>` 저장**. 다음 recombination에서 template hash mismatch 감지 시 Stage 3 Preview 에 "Template updated: regenerate affected artifacts? [y/n]" 표시.

**구현**:
```typescript
// src/plasmids/schema/lock-v1.ts 확장 (P-1.1 §2.1)
templateHashes: z.record(z.string()).default({}),

// src/recombination/generators/template-drift-check.ts 신규
export async function detectTemplateDrift(lock: LockFileV1): Promise<DriftResult>
```

**Rationale**: Template을 바꿨는데 artifact 미생성 → 실제 동작과 template 괴리. Hash tracking 으로 투명한 drift 감지.
