**작성일**: 2026-04-23

**이 문서의 역할**: `<name>.md` (source, user-owned, immutable by system) + `<name>.compiled.lock.json` (system-owned) 2-file model의 규칙 + lock schema v1을 짧게 재확인한다.

---

## 0. TL;DR

- `.md` = SOURCE. 사용자만 편집. System writes 0.
- `.compiled.lock.json` = SYSTEM STATE. Recombination만 편집. User 편집 금지 (경고).
- 두 파일은 **같은 이름 stem** 공유 (`owasp-gate.md` ↔ `owasp-gate.compiled.lock.json`)
- I-1 위반 감지: system이 `.md` 수정 시 에러 throw (dev-time assert).
- Lock file schema v1 (`schemaVersion: 1`) — 확장은 minor version bump.

---

## 1. 두 파일의 역할 분리

| 파일 | 소유자 | 쓰기 권한 | 읽기 시점 | 내용 |
|-----|-------|----------|---------|-----|
| `<name>.md` | 사용자 | 사용자 only (I-1 강제) | compile-time only (I-8) | frontmatter + body (intent 선언) |
| `<name>.compiled.lock.json` | System | `/recombination`, `/cure` | compile-time only (I-8) | compiled artifacts plan + cache 메타 |

### 1.1 I-1 준수 원칙

- Recombination / Cure / Interpreter / Compressor / Reorganizer 모두 `.md` 파일 쓰기 금지
- `src/plasmids/loader.ts` 가 `.md` 파일을 읽고 parse 결과만 반환. 내부에서 수정 안 함.
- I-1 위반을 dev-time에 잡기 위해 `src/plasmids/safe-fs.ts` (신규) wrapper:

```typescript
export async function writeToPlasmidMd(path: string, content: string): Promise<never> {
  throw new Error(`I-1 VIOLATION: attempted write to ${path}. Only user may edit plasmid .md files.`);
}
export async function writeToLockFile(path: string, content: object): Promise<void> {
  if (!path.endsWith(".compiled.lock.json")) {
    throw new Error(`I-1 VIOLATION: ${path} is not a .compiled.lock.json path`);
  }
  await atomicWrite(path, JSON.stringify(content, null, 2));
}
```

모든 persistence stage에서 이 wrapper 경유 → `.md`에 쓰는 코드 path가 존재하지 않음.

### 1.2 Sync 규칙

- `.md` 변경 → next `/recombination`이 lock을 재생성 (cache invalidate)
- `.md` 삭제 → next `/recombination` 또는 `/cure`가 orphan lock 감지 + 사용자 확인 후 lock 삭제
- Lock만 있고 `.md` 없으면 orphan 경고 (`.dhelix/plasmids/README.md`에 cleanup 안내)

## 2. Lock File Schema v1 확정

```typescript
// src/plasmids/schema/lock-v1.ts
import { z } from "zod";

export const artifactSpecSchema = z.object({
  kind: z.enum(["hook","rule","skill","command","agent","harness","prompt-section"]),
  path: z.string(),
  templateId: z.string().optional(),
  vars: z.record(z.unknown()).optional(),
  generatedHash: z.string(),    // for cure integrity check
});

export const lockFileV1Schema = z.object({
  schemaVersion: z.literal(1),
  plasmidName: z.string(),
  plasmidHash: z.string(),                         // source .md content hash
  compiledAt: z.string().datetime(),
  compilerVersion: z.string(),                     // "dhelix@0.4.0"
  interpreterVersion: z.string(),                  // interpreter prompt version
  llmModelId: z.string(),
  llmModelDigest: z.string().optional(),           // P-1.18 drift detection
  artifacts: z.array(artifactSpecSchema),
  compressedSummary: z.string(),                   // 50-150 tokens, Layer B output
  summaryHash: z.string(),
  userApproved: z.boolean(),
  userApprovedAt: z.string().datetime().optional(),
  generatedFiles: z.array(z.string()),             // cure가 삭제할 경로 목록
  invalidatedBy: z.array(z.enum(["plasmid-hash", "model-drift", "schema-version", "user-cure"])).default([]),
});

export type LockFileV1 = z.infer<typeof lockFileV1Schema>;
```

### 2.1 Schema Migration (향후 v2)

```typescript
// src/plasmids/schema/lock-migrations.ts
export function migrateLockFile(raw: unknown): LockFileV1 {
  const obj = raw as Record<string, unknown>;
  const version = obj.schemaVersion ?? 0;
  switch (version) {
    case 0: return migrateV0ToV1(obj);
    case 1: return lockFileV1Schema.parse(obj);
    default: throw new Error(`Unknown lock schema version: ${version}`);
  }
}
```

## 5. Resolved Questions (v0.2)

P-1.1은 v0.1 작성 시 별도 Open Questions 섹션 없이 명세 확정. v0.2 승격 과정에서 발견한 추가 결정:

### Q1 — Orphan lock 자동 cleanup 임계값

**결정**: **24시간 기준**. 그 이상 경과된 orphan `.compiled.lock.json` (source `.md` 삭제된 상태) 은 `/plasmid list` 실행 시 자동 감지 + 사용자 1회 확인 후 제거.

**Rationale**: Source 삭제 직후 즉시 orphan cleanup은 실수 복구 기회 박탈 (git 없을 경우). 24h 유예는 사용자가 "어 내가 왜 지웠지" 회복 충분.

**구현**: `src/plasmids/orphan-detector.ts` 신규 (20 LOC). `/plasmid list` handler 에서 호출.

### Q2 — Lock schema version 상향 정책

**결정**: `schemaVersion: 2` 필요 시 **자동 migration on read** (`migrateLockFile`) + 쓰기 시 항상 최신 schema. 과거 버전 lock은 **never written back** (forward-only).

**Rationale**: Backwards compat 복잡성 감소. Migration은 순수 함수라 side-effect 없음.

### Q3 — `I-1 VIOLATION` 에러 발생 시 런타임 처리

**결정**: Development (`NODE_ENV !== "production"`) 에서는 throw, production 에서는 **log + graceful degrade** (해당 쓰기만 생략).

**Rationale**: I-1 violation 은 개발 시 코드 결함. Production 까지 가면 사용자 작업 중단 막아야 함 (silent skip + telemetry).
