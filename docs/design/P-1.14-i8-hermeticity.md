## 0. TL;DR

- **Invariant I-8**: runtime agent loop는 `.dhelix/plasmids/**`, `.dhelix/recombination/**`, `*.compiled.lock.json`을 읽지 못한다.
- **3층 방어**: (1) 인스트럭션 로더 제외 (2) 도구 guardrail (3) telemetry.
- **경계**: tool 호출 경로만 차단. command handler(`/plasmid`, `/recombination`)는 `fs.readFile` 직접 사용으로 우회.
- **수정 대상 파일**: 6개. 신규 파일: 1개 (`src/instructions/runtime-blocklist.ts`).
- **Metric**: `plasmid.runtime_access_attempt{tool_name, pattern}` — 정상 상태는 **항상 0**.

## 2. 차단 대상 패턴 확정

**파일**: `src/instructions/runtime-blocklist.ts` (신규)

```typescript
/**
 * Runtime agent loop에서 접근이 금지된 경로 패턴.
 * I-8 (Compile-Runtime Hermeticity) 강제.
 *
 * /plasmid, /recombination command handler는 agent-loop 밖에서
 * fs.readFile을 직접 호출하므로 이 블록리스트의 영향을 받지 않는다.
 */
export const RUNTIME_BLOCKED_PATTERNS = [
  // .dhelix/plasmids/ 하위 전체 (소스)
  /(^|[/\\])\.dhelix[/\\]plasmids([/\\]|$)/,
  // .dhelix/recombination/ 하위 전체 (시스템 상태 + transcript)
  /(^|[/\\])\.dhelix[/\\]recombination([/\\]|$)/,
  // 파일명 접미사 기반 (어디에 있든 차단)
  /\.compiled\.lock\.json$/,
] as const;

export function isRuntimeBlocked(filePath: string): {
  readonly blocked: boolean;
  readonly matchedPattern?: string;
} {
  const normalized = filePath.replace(/\\/g, "/");
  for (const pattern of RUNTIME_BLOCKED_PATTERNS) {
    if (pattern.test(normalized)) {
      return { blocked: true, matchedPattern: pattern.source };
    }
  }
  return { blocked: false };
}
```

**설계 결정**:
- 패턴은 cross-platform (Windows `\\`와 POSIX `/` 모두). `normalizePath()`로 통일 후 검사.
- `@import` resolver와 path-matcher도 이 함수를 통과해야 함.
- 전역 `~/.dhelix/plasmids/`도 차단 (runtime은 global plasmid도 못 읽음, recombination에서만 해석).

### 3.2 변경 내용

```typescript
// import 추가
import { isRuntimeBlocked } from "./runtime-blocklist.js";
import { emitRuntimeAccessAttempt } from "../telemetry/plasmid-guard.js";

async function safeReadFile(filePath: string): Promise<string> {
  // I-8 강제: runtime loader는 plasmid/recombination 영역을 읽지 않음
  const check = isRuntimeBlocked(filePath);
  if (check.blocked) {
    emitRuntimeAccessAttempt({
      origin: "instruction_loader",
      path: filePath,
      pattern: check.matchedPattern!,
    });
    return "";  // silent drop — loader는 에러 throw 대신 빈 문자열 (기존 동작 유지)
  }
  try {
    const resolvedPath = await realpath(filePath);
    return await readFile(resolvedPath, "utf-8");
  } catch { return ""; }
}
```

### 3.3 추가 제외 지점

- **`parseInstructions` → `@import`**: `src/instructions/parser.ts` 의 import resolver가 임의 경로를 읽을 수 있음. 동일 `isRuntimeBlocked` 체크 경유 후 "(import blocked by I-8)" 주석 inline.
- **`path-matcher.ts`의 `collectMatchingContent`**: rule 파일 자체가 plasmid 경로 하위에 있을 수 없으므로 loader가 rule 디렉토리 스캔 단계에서 제외. 현재 `.dhelix/rules/` 만 스캔하므로 간접적으로 안전하지만, `glob` 패턴이 `**`를 쓰는 경로 규칙(`paths: ["**"]`)은 runtime 파일 매칭으로 확장될 수 있음 → `collectMatchingContent` 결과에서 blocked 경로 제거.

**Done 기준 (Layer 1)**: loader unit test에서 `.dhelix/plasmids/a.md` 지정 시 빈 문자열 반환 + telemetry event 1회 발행.

## 4. Layer 2 — Tool-Level Guardrail

### 4.1 수정 대상 매트릭스

| 도구 | 파일 | 수정 지점 | 차단 대상 인자 |
|-----|------|---------|--------------|
| `file_read` | `src/tools/definitions/file-read.ts` | `execute()` 첫 줄 (line 578) | `params.path` |
| `file_write` | `src/tools/definitions/file-write.ts` | `execute()` 첫 줄 | `params.path` |
| `file_edit` | `src/tools/definitions/file-edit.ts` | `execute()` 첫 줄 | `params.path` |
| `glob_search` | `src/tools/definitions/glob.ts` | 결과 필터링 | `result.paths[]` |
| `grep_search` / `search_content` | `src/tools/definitions/grep.ts` | 결과 필터링 | `result.matches[].path` |
| `bash` | `src/tools/definitions/bash.ts` | guardrail pre-check | `params.command` — path 추출 후 검증 |

**파일 편집 3종 (read/write/edit)**: 직접 인자 검증.
**검색 도구 2종 (glob/grep)**: 결과에서 blocked 경로를 제외 (enumerate 차단). 결과가 비어도 에러 아님.
**bash**: 제한적. `cat .dhelix/plasmids/foo.md`, `rg foo .dhelix/plasmids` 같은 명령을 토큰 단위로 스캔. 복잡한 파이프나 eval은 놓칠 수 있으나, **bash의 목표는 best-effort 추가 방어**이지 유일 guardrail이 아님 (L1이 최후 보루).

### 4.2 통합 지점 선택 — preflight vs tool 내부

현재 `src/tools/pipeline/preflight.ts`의 `guardrail-scan` 검사가 `applyInputGuardrails(toolName, args, workingDirectory)`를 호출 (line 125). **여기에 plasmid-blocklist를 추가하는 편이 개별 도구 수정보다 낫다** — 새 file-I/O 도구가 추가될 때도 자동 적용됨.

**결정**: `src/guardrails/plasmid-blocklist.ts` (신규) 를 만들고, `src/guardrails/index.ts`의 `applyInputGuardrails` 에서 체인 호출.

```typescript
// src/guardrails/plasmid-blocklist.ts (신규)
import { isRuntimeBlocked } from "../instructions/runtime-blocklist.js";
import { emitRuntimeAccessAttempt } from "../telemetry/plasmid-guard.js";

/** 이 도구들에 대해서만 path 인자를 검사 */
const PATH_TOOL_ARGS: Record<string, readonly string[]> = {
  file_read: ["path"],
  file_write: ["path"],
  file_edit: ["path"],
  // glob_search / grep_search는 결과 필터링에서 처리 (여기선 X)
};

export function checkPlasmidBlocklist(
  toolName: string,
  args: Record<string, unknown>,
): { passed: boolean; reason?: string } {
  const argNames = PATH_TOOL_ARGS[toolName];
  if (!argNames) return { passed: true };
  for (const name of argNames) {
    const value = args[name];
    if (typeof value !== "string") continue;
    const check = isRuntimeBlocked(value);
    if (check.blocked) {
      emitRuntimeAccessAttempt({
        origin: "tool_guardrail",
        toolName,
        path: value,
        pattern: check.matchedPattern!,
      });
      return {
        passed: false,
        reason:
          "PLASMID_RUNTIME_ACCESS_DENIED: Plasmid sources are compile-time only. " +
          "Use compiled artifacts instead:\n" +
          "  - Active rules: .dhelix/prompt-sections/generated/\n" +
          "  - Skills: .dhelix/skills/\n" +
          "  - Hooks: .dhelix/hooks/\n" +
          "To modify intent: /plasmid edit <name> then /recombination",
      };
    }
  }
  return { passed: true };
}
```

### 4.3 grep/glob 결과 필터링

검색 도구는 인자 차단이 아니라 **결과 필터링**. 각 도구의 `execute()` 끝에서:

```typescript
const filtered = results.filter(r => !isRuntimeBlocked(r.path).blocked);
if (filtered.length !== results.length) {
  // blocked 개수만 metric, 결과엔 언급 안 함 (information leak 방지)
  emitRuntimeAccessAttempt({
    origin: "search_filter",
    toolName: "glob_search",
    count: results.length - filtered.length,
  });
}
```

### 4.4 에러 반환 포맷

`ToolCallResult` 의 `isError: true` + `output`에 `PLASMID_RUNTIME_ACCESS_DENIED` 접두. 에러 코드는 PRD §10.3 카탈로그에 이미 존재.

**Done 기준 (Layer 2)**: `file_read {path: ".dhelix/plasmids/foo.md"}` 호출 시 preflight에서 `rejected`로 분류되고, 메시지에 우회 경로(`.dhelix/prompt-sections/generated/`) 포함.

## 5. Layer 3 — Telemetry

### 5.1 신규 파일

`src/telemetry/plasmid-guard.ts`:

```typescript
import { eventBuffer } from "./events.js";

export interface PlasmidGuardEvent {
  readonly timestamp: string;
  readonly type: "plasmid_runtime_access_attempt";
  readonly origin: "instruction_loader" | "tool_guardrail" | "search_filter";
  readonly toolName?: string;
  readonly path?: string;
  readonly pattern?: string;
  readonly count?: number;  // search_filter 전용
  readonly sessionId?: string;
}

export function emitRuntimeAccessAttempt(
  partial: Omit<PlasmidGuardEvent, "timestamp" | "type">,
): void {
  eventBuffer.record({
    timestamp: new Date().toISOString(),
    type: "plasmid_runtime_access_attempt",
    ...partial,
  });
}
```

### 5.2 Metric 매핑

| Metric | 레이블 | 정상 값 | 알람 |
|-------|------|-------|-----|
| `plasmid.runtime_access_attempt` | `origin`, `tool_name`, `pattern` | **0** | 1회 이상 시 prompt 검토 |
| `plasmid.runtime_access_blocked_total` | (cumulative) | 0 증가 | week-over-week 증가 시 조사 |

OTLP exporter (`src/telemetry/otel-exporter.ts`)가 event 타입 `plasmid_runtime_access_attempt`를 자동으로 metric으로 변환. 기존 패턴 재사용 (현재 `tool_execution` 등).

### 5.3 Dashboard 의도

PRD §12.3 내부 측정의 "`plasmid.runtime_access_attempt = 0` 불변식 확인"을 Plasmid Health panel에 시계열 그래프 + 경보 룰로 표현 (구현은 Phase 3+).

## 6. Exception — Command Handler

archive/hardening §1.5 원칙:

> 경계는 "dhelix 프로세스 전체"가 아니라 **"agent-loop의 tool 호출 경로"** 다.

### 6.1 구현 규칙

- `src/commands/plasmid.ts`, `src/commands/recombination.ts`, `src/commands/cure.ts` (Phase 1+ 신규)는 **직접 `fs.readFile` 호출**. `executeFileRead` tool을 **경유하지 않음**.

- Guardrail은 `executeFileRead` / preflight에만 걸려 있고 `fs.readFile`에는 없다 → 자연스럽게 우회됨.

### 6.2 검증

- 리뷰 시 command handler에서 `executeFileRead` 호출 없는지 확인.
- 통합 테스트에서 `/plasmid list` 가 .dhelix/plasmids/ 열람 성공해야 함.

## 10. Resolved Questions (v0.2 확정)

### Q1 — `bash` 토큰 스캔 범위

**결정**: best-effort 스캔. `bash` tool의 `params.command` 문자열을 space-split 후 **화이트리스트 명령어** 뒤의 **첫 positional argument**만 검사.

**화이트리스트**: `cat, less, more, head, tail, rg, grep, fd, find, xxd, bat, awk, sed` (파일 경로를 첫 인자로 받는 일반 읽기 도구).

**허용되는 우회 (의도적 수용)**:
- 파이프 뒤 명령 (`echo foo | cat .dhelix/plasmids/bar.md`)
- 쉘 변수 (`FILE=.dhelix/plasmids/bar.md && cat $FILE`)
- eval / base64 난독화
- 새 탐지 도구 (최근 릴리스된 CLI)

**근거**: L1 (loader exclusion)이 최종 방어선. L2는 **agent가 자발적으로 plasmid를 읽으려는 직관적 시도**를 잡는 게 목표이지, 완전 sandbox가 아님. 우회 가능성은 telemetry로 감지 + 프롬프트 개선으로 대응.

### Q2 — `glob_search` 결과 silent-drop vs. 투명 경고

**결정**: **silent drop + telemetry 카운터**. 결과에서 blocked 경로를 제거하되 사용자/LLM에 **"N개 파일이 필터링됨" 같은 언급 없이** 반환.

**근거**: 투명 경고는 information leak — "plasmid가 몇 개 있는지, 어느 이름인지" 힌트가 agent에 노출됨. telemetry 이벤트 `origin: "search_filter"` + `count` 필드로 내부 감사만.

### Q3 — 글로벌 blocklist에 `~/.dhelix/plasmids/` 포함?

**결정**: **포함**. 패턴 확장 필요.

`RUNTIME_BLOCKED_PATTERNS` 첫 번째 정규식을 `(^|[/\\])\.dhelix[/\\]plasmids([/\\]|$)` 로 유지하고, `isRuntimeBlocked` 에서 `path`를 먼저 `normalizePath`로 통일한 뒤 **homeDir prefix 절대경로**도 검사:

```typescript
export function isRuntimeBlocked(filePath: string): {...} {
  const normalized = normalizePath(filePath);
  // 1) 기존 상대 경로 패턴 검사
  for (const p of RUNTIME_BLOCKED_PATTERNS) { if (p.test(normalized)) return {...}; }
  // 2) 전역 경로도 차단 — homeDir resolve 후 재검사
  const home = normalizePath(process.env.HOME ?? process.env.USERPROFILE ?? "");
  if (home && normalized.startsWith(home + "/.dhelix/plasmids/")) return { blocked: true, matchedPattern: "global-plasmids" };
  if (home && normalized.startsWith(home + "/.dhelix/recombination/")) return { blocked: true, matchedPattern: "global-recombination" };
  return { blocked: false };
}
```

**근거**: PRD §6.1.1 "전역 plasmid (`~/.dhelix/plasmids/`) 도 compile-time only". runtime은 project든 global이든 plasmid 소스를 보지 못함.

### Q4 — `@import` 차단 시 표시

**결정**: **HTML 주석 한 줄 inline 삽입**.

`src/instructions/parser.ts` 의 `@import` resolver에서 차단 시:
```typescript
if (isRuntimeBlocked(importPath).blocked) {
  emitRuntimeAccessAttempt({ origin: "instruction_loader", path: importPath, pattern: "..." });
  return `<!-- I-8 blocked: ${basename(importPath)} (plasmid source is compile-time only) -->`;
}
```

**근거**: 완전 빈 문자열이면 파싱 결과가 "왜 비어있는지" 디버그 불가. 짧은 주석은 information leak 최소 (파일명 basename만 노출) + 디버깅 용이. LLM에 주석 내용이 전달돼도 경로 / 내용 복원 불가능.
