## 1. config.json 스키마

### 1.1 전체 스키마

```typescript
// src/config/schema.ts (확장)
import { z } from "zod";

const modelSpecSchema = z.union([
  z.string(),    // simple: "llama3.1:8b"
  z.object({
    primary: z.string(),
    fallbackOnSchemaFailure: z.string().optional(),   // cloud cascade
    reorgFallback: z.string().optional(),             // 특정 stage만
  }),
]);

export const modelsConfigSchema = z.object({
  runtime: modelSpecSchema.default("claude-opus-4-7"),
  recombination: modelSpecSchema.optional(),          // 없으면 runtime 사용
  validation: modelSpecSchema.optional(),             // 없으면 runtime 사용
});

export const cloudConfigSchema = z.object({
  cascadeEnabled: z.boolean().default(false),
  promptBeforeCall: z.boolean().default(true),
  auditLog: z.boolean().default(true),
}).default({});

// 기존 config 스키마에 추가
export const dhelixConfigSchema = z.object({
  // ... 기존 필드
  models: modelsConfigSchema.default({ runtime: "claude-opus-4-7" }),
  cloud: cloudConfigSchema,
  // ...
});

export type ModelsConfig = z.infer<typeof modelsConfigSchema>;
export type CloudConfig = z.infer<typeof cloudConfigSchema>;
```

### 1.2 Resolve 함수

```typescript
// src/config/resolve-model.ts (신규)
export interface ResolvedModel {
  readonly primary: string;
  readonly fallback?: string;
  readonly reorgFallback?: string;
}

export function resolveModel(
  config: ModelsConfig,
  role: "runtime" | "recombination" | "validation",
): ResolvedModel {
  const spec = config[role] ?? config.runtime;   // 미설정 시 runtime fallback
  if (typeof spec === "string") return { primary: spec };
  return {
    primary: spec.primary,
    fallback: spec.fallbackOnSchemaFailure,
    reorgFallback: spec.reorgFallback,
  };
}
```

## 2. 예시 Config 3종

### 2.1 All-Local (Privacy-first)

```json
{
  "models": {
    "runtime": "llama3.1:8b",
    "recombination": "qwen2.5-coder:32b",
    "validation": "llama3.1:8b"
  },
  "cloud": {
    "cascadeEnabled": false
  }
}
```

**특징**:
- Runtime은 빠른 작은 모델 (매 turn 응답)
- Recombination은 드물지만 품질 중요 → 큰 local 모델
- Validation은 runtime과 동일 → "실제 경험 반영"
- Cloud 연결 **없음** (네트워크 offline 가능)

### 2.2 Hybrid (Cost-Optimized)

```json
{
  "models": {
    "runtime": "llama3.1:8b",
    "recombination": {
      "primary": "llama3.1:8b",
      "fallbackOnSchemaFailure": "claude-haiku-4-5",
      "reorgFallback": "claude-haiku-4-5"
    },
    "validation": "llama3.1:8b"
  },
  "cloud": {
    "cascadeEnabled": true,
    "promptBeforeCall": true
  }
}
```

**특징**:
- 모든 stage에서 local 우선
- Schema 실패 시 cheap cloud (Haiku)로 cascade
- 매 cloud call 사용자 확인 (`promptBeforeCall: true`)
- `privacy: local-only` plasmid는 여전히 cascade 안 함

### 2.3 All-Cloud (Max Quality)

```json
{
  "models": {
    "runtime": "claude-opus-4-7",
    "recombination": "claude-opus-4-7",
    "validation": "claude-haiku-4-5"
  },
  "cloud": {
    "cascadeEnabled": false
  }
}
```

**특징**:
- 전부 cloud, cascade 불필요
- Validation만 cheap model (비용 절감)

## 3. Cloud Cascade 흐름

### 3.1 발동 조건

다음 모두 충족 시에만 cloud cascade 실행:

1. `cloud.cascadeEnabled === true`
2. 현재 role의 `fallbackOnSchemaFailure` 또는 `reorgFallback` 설정됨
3. local LLM이 schema failure (JSON/XML parse fail + retry 모두 실패) 또는 reorg fallback 조건
4. 현재 compile 중인 plasmid 중 어느 것도 `privacy: local-only` 아님

### 3.2 User Consent UI

```
⚠ Cloud Cascade Requested
─────────────────────────────
Stage:  Constitution Reorganizer (reorg)
Local:  llama3.1:8b (failed 2× JSON parse)
Cloud:  claude-haiku-4-5

Privacy impact:
  - The following plasmid bodies will be sent to Anthropic API:
    • owasp-gate
    • ddd-review
  - (no plasmid is marked privacy: local-only)

Actions:
  [y] Send to cloud (one-time for this stage)
  [a] Always allow for this session
  [d] Deny — use deterministic fallback instead
  [i] Inspect what exactly will be sent
```

### 3.3 Audit Log

`cloud.auditLog: true` (default) → 모든 cloud cascade를 `.dhelix/recombination/audit.log` 에 append:

```
2026-04-23T10:32:01Z CLOUD_CALL stage=reorg model=claude-haiku-4-5 plasmids=[owasp-gate,ddd-review] tokens=1240 cost=$0.0031 reason=local-llm-json-fail
```

## 4. `privacy: local-only` 강제

### 4.1 스키마

Plasmid frontmatter:
```yaml
---
name: proprietary-auth
privacy: local-only
---
```

`src/plasmids/schema/v1.ts`에 추가:
```typescript
privacy: z.enum(["local-only", "default"]).default("default"),
```

### 4.2 Enforcement Point

**모든 LLM call 직전** 검사:

```typescript
// src/recombination/llm-gate.ts (신규)
export function assertCloudEligible(
  llmTarget: "cloud" | "local",
  activePlasmids: readonly Plasmid[],
): void {
  if (llmTarget === "local") return;   // local은 항상 OK
  const privateSet = activePlasmids.filter(p => p.privacy === "local-only");
  if (privateSet.length > 0) {
    throw new PrivacyViolationError({
      code: "PRIVACY_CLOUD_BLOCKED",
      message: `Cannot send to cloud: ${privateSet.length} plasmid(s) marked privacy: local-only`,
      plasmids: privateSet.map(p => p.name),
    });
  }
}
```

**호출 위치**:
- `llm-gate.assertCloudEligible` 는 recombination 중 **cloud LLM 호출 전 항상** 실행
- Stage 2a interpreter, 2c compression Layer B, 2d reorg LLM, 6 validation LLM judge 각각

### 4.3 Error Surface

`PrivacyViolationError` 발생 시:
- Abort current stage
- `privacy.cloud_blocked_total` metric + 1
- 사용자에게 표시:
  ```
  ⛔ Privacy violation blocked
  Plasmid(s) marked 'privacy: local-only':
    - proprietary-auth

  Options:
  [a] Continue with local LLM only (may fail for unreliable models)
  [b] Temporarily reclassify plasmid (requires editing frontmatter)
  [c] Abort recombination
  ```

**Reclassify** `[b]` 시 plasmid .md 직접 편집 후 `/recombination` 재시도 (자동 수정 안 함 — 사용자 책임).

## 5. Validation-Specific 분리

`models.validation`이 runtime과 다를 때의 의미:

### 5.1 왜 분리 가능한가

- Recombination 결과가 "validation 모델 기준으로 pass" → 실제 runtime 모델과 괴리 가능
- 같은 모델로 validate 하는 것이 원칙

### 5.2 분리 허용 시나리오

- CI 환경에서 validation만 cloud로 (fast, deterministic) + runtime은 local
- Validation에 큰 cloud 모델 (LLM judge), runtime은 작은 local
- Regression test 시 golden model 고정

**Warning UX**:
```
⚠ Validation model differs from runtime
  runtime:    llama3.1:8b
  validation: claude-haiku-4-5
  
  Validation results reflect Haiku's behavior, not llama3.1:8b.
  Consider aligning if you're testing llama3.1:8b's actual behavior.
  [y] Continue  [a] Align to runtime
```

## 6. Default Resolution 룰

| Config 상태 | Resolution |
|-----------|------------|
| `models.runtime` 만 설정 | recombination = validation = runtime |
| `models.recombination` 별도 | validation = runtime (runtime fallback) |
| `models.validation` 별도 | runtime이 별도 설정되지 않았으면 runtime = validation |
| 전부 미설정 | runtime = `claude-opus-4-7` (default), 나머지 동일 |

**근본 원칙**: **runtime 이 baseline**. 다른 role은 명시적 override 없으면 runtime 따라감.

## 7. Model Drift 통합 (P-1.18 §4 연동)

Runtime 모델 digest가 변경되면 **validation 재실행 권장**.

```typescript
// Stage 0 preflight
const runtimeDigest = await probeModel(resolveModel(config, "runtime"));
const lastValidationDigest = await readLastValidationDigest();
if (runtimeDigest !== lastValidationDigest) {
  notifyUser({
    level: "warning",
    message: "Runtime 모델이 업데이트되었습니다. Validation 재실행 권장.",
    recommendation: "전체 plasmid re-validation",
    actions: ["re-validate-all", "continue", "abort"],
  });
}
```

## 10. Resolved Questions (v0.2)

### Q1 — `models.runtime` 기본값

**확정**: **v0.4 기본 = `claude-opus-4-7`. Ollama 감지 시 `harness-setup` 에서 `llama3.1:8b` 자동 권장**.

```typescript
// src/config/defaults.ts
export const DEFAULT_RUNTIME_MODEL = "claude-opus-4-7";
```

`harness-setup` 스킬 로직:
```typescript
async function suggestRuntimeModel(): Promise<string> {
  if (await isOllamaRunning()) {
    const models = await listOllamaModels();
    if (models.some(m => m.name === "llama3.1:8b")) {
      return "llama3.1:8b";    // Privacy-first 사용자로 추정
    }
  }
  return DEFAULT_RUNTIME_MODEL;
}
```

**Rationale**: Cloud default = 새 사용자 onboarding 최대 품질 (Anthropic의 신뢰성). Ollama 설정된 환경 = 이미 local LLM 의도 → 자동 suggest.

### Q2 — `privacy: local-only` 부분적 처리 (v0.4 전면 vs Phase 4+ 세분화)

**확정**: **v0.4 = 하나라도 private면 전체 cloud 금지 (보수적). Phase 4+ 에서 per-plasmid 분기 고려**.

**v0.4 로직** (P-1.21 §4.2 `assertCloudEligible`):
```typescript
function assertCloudEligible(llmTarget: "cloud" | "local", activePlasmids: readonly Plasmid[]): void {
  if (llmTarget === "local") return;
  const privateSet = activePlasmids.filter(p => p.privacy === "local-only");
  if (privateSet.length > 0) {
    throw new PrivacyViolationError({ ..., plasmids: privateSet.map(p => p.name) });
  }
}
```

**Phase 4+ 검토 (per-plasmid partition)**:
- Private plasmid 의 summary/body 만 local LLM 으로 처리
- Non-private plasmid 는 cloud 로 cascade
- 기술적: Stage 2a interpreter를 plasmid별로 나눠 호출
- 위험: orchestration 복잡도 폭증 + accidental cross-contamination

**Rationale**: v0.4 안전성 우선 — privacy violation 이 무엇보다 치명적. 복잡한 partition 은 경험 데이터 축적 후.

### Q3 — `fallbackOnSchemaFailure` vs `reorgFallback` 우선

**확정**: **Stage-specific (`reorgFallback`) 가 generic (`fallbackOnSchemaFailure`) 보다 우선**.

```typescript
// src/config/resolve-model.ts
function resolveFallbackForStage(config: ModelsConfig, stage: Stage): string | undefined {
  const specific = config.recombination?.reorgFallback;    // stage-specific
  if (stage === "reorg" && specific) return specific;
  return config.recombination?.fallbackOnSchemaFailure;    // generic
}
```

**Rationale**: Stage-specific 설정은 사용자의 더 의도적 선택 — 덮어쓰기 우선. Generic 은 fallback of fallback.

### Q4 — `models.validation` cloud + private plasmid 충돌

**확정**: **해당 private plasmid 는 validation skip + warning. 사용자는 local validation 모델로 별도 실행 안내**.

UX:
```
⚠ Plasmid 'proprietary-auth' (privacy: local-only) cannot be validated with cloud model claude-haiku-4-5.
  Validation will skip this plasmid.
  
  To validate separately:
    $ /recombination --validate-only --plasmid proprietary-auth --model llama3.1:8b
```

**Rationale**: Validation 자체 차단은 UX 최악 (사용자가 대안 몰라 진행 불가). Skip + 명확 안내가 진행 가능하게.
