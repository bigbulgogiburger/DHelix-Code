## 0. TL;DR

- Foundational tier는 "도전 비용이 높지만 불가능하지 않은" plasmid (Popper 정합)
- `challengeable-by` 옵션 5개 (justification, audit, cooldown, team consensus 등)
- Challenge flow 3 options: **override** (this run only) / **amend** (수정) / **revoke** (삭제)
- `.dhelix/governance/challenges.log` append-only, JSONL
- L4 validation threshold 95% 강제 (일반 plasmid는 60-70%)

tier: foundational | policy | tactical | agent-proposed

## 2. `challengeable-by` Schema

```typescript
// src/plasmids/schema/v1.ts 확장
export const challengeableBySchema = z.object({
  "require-justification": z.boolean().default(true),
  "min-justification-length": z.number().int().min(0).default(50),
  "audit-log": z.boolean().default(true),
  "require-cooldown": z.string().regex(/^\d+[hdw]$/).default("24h"),   // "24h", "3d", "1w"

  // Team 환경 (Phase 5+)
  "require-team-consensus": z.boolean().default(false),
  "min-approvers": z.number().int().min(1).default(1),
  "approver-roles": z.array(z.string()).optional(),   // ["tech-lead", "security"]
}).optional();

export const tierSchema = z.enum(["foundational", "policy", "tactical", "agent-proposed"]).default("policy");
```

### 2.1 Validation 룰

- `tier: foundational` 이면 `challengeable-by` **필수** (Zod refine)
- `min-justification-length` < 20 이면 Zod error — "너무 낮은 bar는 foundational 취지 훼손"
- `require-cooldown` 파싱: `h`(시간)/`d`(일)/`w`(주). 최소 `1h`

```typescript
plasmidExtensionsSchema.superRefine((data, ctx) => {
  if (data.tier === "foundational" && !data["challengeable-by"]) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: "tier: foundational requires 'challengeable-by' field",
    });
  }
});
```

## 3. Challenge Flow — 3 Options

### 3.1 UX 진입

```
$ /plasmid challenge core-values

🧬 Challenging foundational plasmid: core-values
───────────────────────────────────────────────
Current state:
  tier: foundational
  priority: high
  active: true
  last-validated: 2026-04-21 (L4 100% pass)

⚠ This is a foundational plasmid. Challenge requires:
  - justification (min 50 chars)
  - 24h cooldown enforced
  - audit log entry

Choose action:
  [1] Override this run only (keep plasmid active, skip for one recombination)
  [2] Amend (modify plasmid content — requires challenge ceremony)
  [3] Revoke (deactivate or delete — strongest action)
  [c] Cancel

> 1
```

### 3.2 Option 1 — Override (one-shot)

```
Override rationale (min 50 chars):
> The current owasp-gate change needs foundational:core-values temporarily
  overridden because the legacy migration script intentionally uses raw SQL
  that would be flagged by "sql-parameterized-only" — reviewed with team.

Saving challenge...
  ✓ .dhelix/governance/challenges.log (append)
  ✓ Recombination session: skip core-values for next /recombination only
  ✓ Audit timestamp: 2026-04-23T10:45:12Z

This override expires after 1 recombination run.
```

**효과**:
- 다음 `/recombination` 1회에 한해 해당 plasmid를 **비활성 취급** (artifact 생성 스킵, prompt section 미포함)
- 그 이후 자동으로 원래 상태 복귀
- Audit log 엔트리 필수

### 3.3 Option 2 — Amend (수정)

```
Amendment rationale (min 50 chars):
> Extending core-values to explicitly include 'reversibility requirement'
  which was discussed in 2026-04-18 team retro.

Cooldown check: 25h 3m since last challenge ✓ (cooldown: 24h)

Opening .dhelix/plasmids/core-values.md in $EDITOR...

(사용자 편집)

Validation:
  ✓ Zod schema still valid
  ✓ tier still foundational
  ✓ challengeable-by preserved
  ⚠ Note: foundational plasmid change triggers mandatory re-validation of 
    all affected plasmids in next /recombination

Save? [y/n]
> y
  ✓ core-values.md updated (hash: def456... → abc789...)
  ✓ challenges.log entry: type=amend
  ✓ Cooldown restarted (next challenge: 2026-04-24T11:45Z earliest)
```

### 3.4 Option 3 — Revoke (삭제)

가장 강한 action.

```
Revoke rationale (min 50 chars):
> Team decided core-values was too abstract. Replacing with 3 specific
  foundational plasmids: accuracy-first, user-consent, evidence-backed.

Additional ceremony:
  ⚠ Revoking a foundational plasmid affects all dependent plasmids.
  
  Dependents (3):
    - secure-commit-gate (extends: core-values)
    - review-conduct (compatible-with: core-values)
    - foundation-test-coverage (conflicts-with: core-values)

  Actions for dependents:
    [k] Keep them (may cause validation failures)
    [o] Mark orphaned (must resolve before next recombination)
    [r] Revoke dependents as well

> o

Final confirmation:
  Revoke 'core-values' (tier: foundational)? This will:
    - move .dhelix/plasmids/core-values.md to archive/
    - mark 3 dependents as orphaned
    - create audit entry
  Type 'REVOKE core-values' to confirm:
> REVOKE core-values

  ✓ Moved to .dhelix/plasmids/archive/core-values-2026-04-23T10-45Z.md
  ✓ challenges.log entry: type=revoke, dependents=[orphaned]
  ✓ Cooldown restarted
  ⚠ Next /recombination will flag 3 orphaned plasmids for resolution.
```

## 4. Cooldown 강제

### 4.1 알고리즘

```typescript
// src/plasmids/governance/cooldown.ts
export async function checkCooldown(
  plasmid: Plasmid,
  action: "override" | "amend" | "revoke",
): Promise<{ ok: true } | { ok: false; waitUntil: Date; remainingMs: number }> {
  const log = await readChallengesLog();
  const lastEntry = log.find(e => e.plasmidName === plasmid.name);
  if (!lastEntry) return { ok: true };

  const cooldownMs = parseCooldown(plasmid["challengeable-by"]?.["require-cooldown"] ?? "24h");
  const waitUntil = new Date(new Date(lastEntry.timestamp).getTime() + cooldownMs);
  const remainingMs = waitUntil.getTime() - Date.now();
  if (remainingMs > 0) return { ok: false, waitUntil, remainingMs };

  return { ok: true };
}

function parseCooldown(s: string): number {
  const match = s.match(/^(\d+)([hdw])$/);
  if (!match) throw new Error(`Invalid cooldown: ${s}`);
  const n = parseInt(match[1], 10);
  const unit = match[2];
  return n * (unit === "h" ? 3600_000 : unit === "d" ? 86_400_000 : 604_800_000);
}
```

### 4.2 Exceptions

| 상황 | Cooldown 적용? |
|-----|--------------|
| Override (option 1) | 적용 안 함 (one-shot, 장기 영향 없음) |
| Amend (option 2) | 적용 |
| Revoke (option 3) | 적용 |
| 동일 plasmid에 대한 2번째 amend | 최근 amend 기준 |
| 다른 foundational plasmid | 독립 (각 plasmid 별 cooldown) |

## 10. Resolved Questions (v0.2)

### Q1 — Team consensus v0.4 scope

**확정**: **v0.4 schema-only (forward-compat). UI는 v0.5+**.

`require-team-consensus`, `min-approvers`, `approver-roles` 필드 Zod schema에 포함하되, v0.4 handler 는 이들 무시 (warn if set: "team consensus requires dhelix v0.5+"). Challenge 는 single-user mode (1인 justification으로 충족).

**Rationale**: Schema 선 확정으로 v0.5 migration 부담 제거. UI 복잡도 (approval tracking, notification) 는 v0.5 에서 별도 설계.

### Q2 — `override` 유효 기간

**확정**: **v0.4 "다음 recombination 1회" only. Session-span override 는 v0.5**.

```typescript
// src/plasmids/governance/challenges-log.ts
interface OverrideEntry {
  plasmidName: string;
  consumedAtRecombination: string | null;   // 소비 후 transcript ID
}
```

Recombination 시 `consumedAtRecombination === null` 인 가장 최근 override entry 소비 (mark consumed).

**Rationale**:
- 여러 recombination 에 걸친 override = 지속 영향 → 그럼 차라리 `/plasmid deactivate` 쓰는게 의도 명확
- 1회 override 는 hot-fix 시나리오 (특정 커밋 긴급 통과) 에 최적

### Q3 — Foundational plasmid 간 충돌 tiebreaker

**확정**: **`created` 타임스탬프 오름차순 (older wins)**. 같은 timestamp 면 name 알파벳 순.

**Rationale**:
- "먼저 확립된 원칙" 은 프로젝트 DNA 의 foundational 성격 — 뒤에 들어온 foundational 이 덮어쓰면 원칙 불안정
- 새 foundational 이 기존과 충돌 의도면 **명시적으로 기존 revoke + 재도입** (challenge ceremony 경유)

**구현** (Phase 2 `conflict-detector`):
```typescript
function resolveFoundationalConflict(a: Plasmid, b: Plasmid): Plasmid {
  if (a.tier !== "foundational" || b.tier !== "foundational") throw new Error(...);
  const tA = Date.parse(a.created ?? "2099-01-01");
  const tB = Date.parse(b.created ?? "2099-01-01");
  if (tA !== tB) return tA < tB ? a : b;
  return a.name.localeCompare(b.name) < 0 ? a : b;
}
```

### Q4 — Challenge 남용 감지

**확정**: **v0.4 metric 계산만, UI 표시는 Phase 6+**.
