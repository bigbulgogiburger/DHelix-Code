# P-1.19 — Pipeline Strategy Selector Matrix 설계서

**작성일**: 2026-04-23
**상태**: **v0.2 — Open Questions 해결 + Exit Criteria 충족** (Phase -1 설계, P-1.14, P-1.18 선행)
**관련**: PRD Part III §34.2 / archive/hardening §4.3 / P-1.18 ModelCapabilities, P-1.13 Compression, P-1.15 Reorg
**구현 시점**: Phase 2 Week 1 (recombination 오케스트레이터의 핵심)

**이 문서의 역할**: 8-stage pipeline의 각 stage가 `ModelCapabilities`에 따라 어떤 전략을 선택하는지 **코드 스텁 수준**으로 매트릭스 확정.

---

## 0. TL;DR

- `selectStrategies(caps): PipelineStrategies` — pure function, cache 가능
- 3 capability tier: **cloud** / **local-large (≥15B)** / **local-small (<15B)**
- 전략 차원 8개: interpreter, compression, reorg, validation volume, validation parallelism, grading tiers, pass thresholds, project profile
- Capability → strategy 매트릭스는 **완전 결정적**. 같은 caps → 같은 strategies.

## 1. API

```typescript
// src/recombination/strategy.ts
import type { ModelCapabilities } from "../llm/model-capabilities.js";

export type InterpreterStrategy = "single-pass" | "chunked" | "field-by-field";
export type CompressionStrategy = "abstractive" | "extractive";
export type ReorgFallback = "llm-only" | "llm-with-xml-fallback" | "llm-with-deterministic-fallback" | "deterministic-only";
export type ValidationVolumeProfile = "standard" | "governed" | "minimal";
export type GradingTier = "deterministic" | "semi" | "llm";
export type ProjectProfileMode = "full-llm" | "llm-summary" | "static-template";

export interface PipelineStrategies {
  readonly interpreter: InterpreterStrategy;
  readonly compression: CompressionStrategy;
  readonly reorgFallback: ReorgFallback;
  readonly validationVolume: ValidationVolumeProfile;
  readonly validationParallelism: number;
  readonly gradingTiers: readonly GradingTier[];
  readonly passThresholds: { L1: number; L2: number; L3: number; L4: number };
  readonly projectProfile: ProjectProfileMode;
  readonly artifactGeneration: "template-and-llm" | "template-only";
  readonly interpreterRetries: number;
}

export function selectStrategies(caps: ModelCapabilities): PipelineStrategies;
```

**설계 불변식**:
- Pure function, side-effect 없음
- 같은 `caps` → 같은 결과 (strategy caching용)
- `caps.isLocal`과 `caps.paramEstimate`만 조합에 사용 (reliableJson은 reorg 차원에서만 분기)

## 2. Capability 분류

```typescript
type CapTier = "cloud" | "local-large" | "local-small";

function classifyCap(caps: ModelCapabilities): CapTier {
  if (!caps.isLocal) return "cloud";
  const p = caps.paramEstimate ?? 999;   // 모르면 큰 쪽으로 보수 가정 (P-1.18 §8 failure mode)
  return p >= 15 ? "local-large" : "local-small";
}
```

**경계 15B 근거**: archive/hardening §13 Graceful Degradation Matrix. 14B 이하 모델 (llama3.1:8b, deepseek-coder:6.7b)은 multi-step tool use + JSON 안정성 약함. 32B+ (qwen2.5-coder:32b, llama3.1:70b)는 GPT-4o 급 품질 근접.

## 3. 전략 매트릭스 (8 차원)

| 차원 | cloud | local-large (≥15B) | local-small (<15B) |
|------|-------|-------------------|-------------------|
| **interpreter** | `single-pass` | `chunked` | `field-by-field` |
| **compression** | `abstractive` (ratio 5:1) | `abstractive` (ratio 3:1) | `extractive` |
| **reorgFallback** | `llm-only` | `llm-with-xml-fallback` | `deterministic-only` |
| **validationVolume** | `standard` (150 cases) | `governed` (50 cases) | `minimal` (20 cases) |
| **validationParallelism** | 10 | 1 | 1 |
| **gradingTiers** | [deterministic, semi, llm] | [deterministic, semi] | [deterministic] |
| **passThresholds** | { L1:0.95, L2:0.80, L3:0.70, L4:0.60 } | { L1:0.90, L2:0.70, L3:0.60, L4:0.50 } | { L1:0.85, L2:0.60, L3:0.50, L4:0.40 } |
| **projectProfile** | `full-llm` | `llm-summary` | `static-template` |
| **artifactGeneration** | `template-and-llm` | `template-and-llm` | `template-only` |
| **interpreterRetries** | 1 | 2 | 3 |

### 3.1 reorgFallback 추가 분기 (reliableJson)

위 매트릭스는 `reliableJson: true`가 기본값. `reliableJson: false` → fallback chain 한 단계 내려감:
- cloud + reliableJson=false (예: GPT-3.5) → `llm-with-xml-fallback`
- local-large + reliableJson=false → `deterministic-only`
- local-small + reliableJson=* → 항상 `deterministic-only`

### 3.2 artifactGeneration 기준

local-small은 LLM이 template slot 채우기도 품질 낮음 → template-only로 고정. slot은 기본값 또는 빈 문자열.

## 4. 참조 구현 (스텁)

```typescript
export function selectStrategies(caps: ModelCapabilities): PipelineStrategies {
  const tier = classifyCap(caps);
  const reliableJson = caps.reliableJson;

  const base: PipelineStrategies = {
    cloud: {
      interpreter: "single-pass",
      compression: "abstractive",
      reorgFallback: reliableJson ? "llm-only" : "llm-with-xml-fallback",
      validationVolume: "standard",
      validationParallelism: 10,
      gradingTiers: ["deterministic", "semi", "llm"],
      passThresholds: { L1: 0.95, L2: 0.80, L3: 0.70, L4: 0.60 },
      projectProfile: "full-llm",
      artifactGeneration: "template-and-llm",
      interpreterRetries: 1,
    },
    "local-large": {
      interpreter: "chunked",
      compression: "abstractive",
      reorgFallback: reliableJson ? "llm-with-xml-fallback" : "deterministic-only",
      validationVolume: "governed",
      validationParallelism: 1,
      gradingTiers: ["deterministic", "semi"],
      passThresholds: { L1: 0.90, L2: 0.70, L3: 0.60, L4: 0.50 },
      projectProfile: "llm-summary",
      artifactGeneration: "template-and-llm",
      interpreterRetries: 2,
    },
    "local-small": {
      interpreter: "field-by-field",
      compression: "extractive",
      reorgFallback: "deterministic-only",
      validationVolume: "minimal",
      validationParallelism: 1,
      gradingTiers: ["deterministic"],
      passThresholds: { L1: 0.85, L2: 0.60, L3: 0.50, L4: 0.40 },
      projectProfile: "static-template",
      artifactGeneration: "template-only",
      interpreterRetries: 3,
    },
  }[tier];

  return base;
}
```

## 10. Resolved Questions (v0.2)

### Q1 — `paramEstimate` 경계 15B 실증 근거 + 상수 분리

**결정**: **15B 유지. 단 상수로 분리하여 POC 실측 후 조정 용이**.

**근거 (공개 benchmark 종합, 2025~2026)**:

| 모델 | HumanEval | MMLU | JSON mode 안정성 | 분류 |
|-----|----------|------|----------------|-----|
| Llama 3.1 8B | 62.8% | 69.4% | 불안정 (retry 필수) | **local-small** |
| DeepSeek Coder 6.7B | 47.6% | 58.1% | 매우 불안정 | **local-small** |
| Qwen 2.5 Coder 7B | 61.6% | 65.2% | 불안정 | **local-small** |
| Phi-3 Medium 14B | 72.3% | 76.6% | 중간 (경계선) | **local-small** (보수적) |
| Qwen 2.5 Coder 32B | 88.4% | 79.3% | 안정 | **local-large** |
| Llama 3.1 70B | 80.5% | 82.0% | 안정 | **local-large** |
| Claude Haiku 4 (cloud) | ~85% | ~78% | 매우 안정 | cloud |

**관측**: 15B 전후에서 **HumanEval 점수 + JSON output 안정성 양쪽 모두 변곡점** (6B→32B 구간에서 20-30% 점프). Phi-3 14B 는 경계선이나 여전히 간헐적 JSON 실패 → 보수적으로 local-small 분류.

**구현 변경 (§2 classifyCap)**:
```typescript
// src/recombination/strategy.ts

/** Capability tier 경계 (B params). Phase 0 POC 실측 후 PR 1개로 조정 가능. */
export const CAP_TIER_THRESHOLD_B = 15;

function classifyCap(caps: ModelCapabilities): CapTier {
  if (!caps.isLocal) return "cloud";
  const p = caps.paramEstimate ?? 999;   // 모르면 보수적으로 큰 쪽 (local-large)
  return p >= CAP_TIER_THRESHOLD_B ? "local-large" : "local-small";
}
```

**Phase 0 POC adjustment protocol**:
- Ollama 참가자 H4 실측 결과가 threshold 어긋나면 **PR 1개로 상수만 변경**
- 예: Phi-3 14B 가 실측에서 qwen:32b 급이면 → `13` 으로 하향
- 상수 변경은 strategy cache key 일부 → 자동 invalidation

**Rationale**:
- 완전 경험적 조정 가능성 유지 + 현 근거로 합리적 출발점 확보
- Hard-coded magic number 를 export 상수로 승격 → 튜닝 가시화

**영향**:
- `src/recombination/strategy.ts` `CAP_TIER_THRESHOLD_B` export
- Telemetry `strategy.tier_classified{tier}` counter (POC 이후 분포 확인)

### Q2 — `validationParallelism` config override

**결정**: **v0.4에 config 필드 포함**. `.dhelix/config.json` `execution.validationParallelism`.

**Schema 확장** (`src/config/schema.ts`):
```typescript
execution: z.object({
  validationParallelism: z.union([
    z.number().int().positive(),
    z.literal("auto"),
  ]).default("auto"),
}).default({}),
```

**Resolution (`src/recombination/strategy.ts` selectStrategies 확장)**:
```typescript
export function selectStrategies(
  caps: ModelCapabilities,
  config?: DhelixConfig,                   // v0.2 추가 선택 파라미터
): PipelineStrategies {
  const base = computeBaseStrategies(caps);
  const userParallelism = config?.execution?.validationParallelism;
  if (userParallelism && userParallelism !== "auto") {
    if (caps.isLocal && userParallelism > LOCAL_PARALLELISM_SAFE_MAX) {
      warnOnce(
        `Local validationParallelism ${userParallelism} > ${LOCAL_PARALLELISM_SAFE_MAX} ` +
        `risks GPU OOM. Proceeding with user value.`,
      );
    }
    return { ...base, validationParallelism: userParallelism };
  }
  return base;
}

const LOCAL_PARALLELISM_SAFE_MAX = 4;   // 8GB VRAM × ~2GB/inference 가정
```

**Rationale**:
- 하드코딩 1은 dual-GPU / high-RAM 사용자 제약
- Cloud 10 도 provider rate limit 넘으면 실패 → user override 필요
- `"auto"` default 로 backward compat 유지
- Local 상한 4 는 GPU memory 일반 가정 (over-subscribe는 사용자 책임)

**영향**:
- `selectStrategies(caps, config?)` 시그니처 확장
- Telemetry `validation.parallelism_actual` gauge (실사용 분포 추적)
- UX: Stage 0 preflight 에서 `Parallelism: 4 (user override, default auto=1)` 표시

### Q3 — `deterministic-only` reorgFallback 캐시 키 cross-ref 확정

**확정**: **P-1.15 §4.3 cacheKey 공식에 `strategies.reorgFallback` 포함** (v0.1 이미 언급, v0.2에서 두 문서 간 일관성 명시).

**최종 cacheKey 공식 (P-1.15 §4.3 확정판)**:
```typescript
const cacheKey = sha256([
  input.plasmidIntentGraph.canonicalJSON,
  input.existingDHELIX.contentHash,
  REORG_PROMPT_VERSION,
  input.modelCapabilities.modelDigest ?? "unknown",
  input.strategies.reorgFallback,          // ← v0.2 명시 포함
].join("\x00"));
```

**효과**:
- Strategy 가 `deterministic-only` 인 run 결과 → 별도 cache bucket
- Cloud 전환 → `llm-only` 로 strategy 변경 → 캐시 miss → 새로 LLM plan 생성 → 별개 bucket 저장
- 환경 복귀 시 각자 bucket hit — 반복 recombination efficient

**Rationale**:
- Strategy 변화는 plan 구조 변화를 내포 (deterministic은 단순 insert/update/remove, LLM은 merge/order 포함 가능)
- 같은 cache key로 교차 사용하면 mode 간 plan 형식 불일치 발생
- Strategy를 key에 포함하면 각 mode의 plan이 독립 저장

**영향**:
- P-1.15 §4.3 와 P-1.19 §10 cross-reference 확립
- Telemetry `reorg.cache_hit_by_fallback{fallback}` counter — 각 bucket 캐시 효과 독립 추적
- Object store (`.dhelix/recombination/objects/`) 에서 같은 plasmid에 대해 최대 3개 hash (deterministic/llm-only/llm-with-fallback) 존재 가능

### Q4 — Cascade OFF에서 local → cloud 중간 전환 UX

**결정**: **Preflight re-run 경로만 허용. Mid-stage 전환 금지**.

**Flow**:
```
1. Stage 0 Preflight 완료 후 사용자 옵션 제시:
   [y] Continue  [c] Switch to cloud (re-run preflight)  [n] Abort

2. [c] 선택 시:
   2a. Privacy check: activePlasmids 중 'privacy: local-only' 존재 → assertCloudEligible throws
   2b. config.models.runtime 임시 session override (in-memory only)
   2c. Stage 0 완전 재실행 (capability probe + strategy recompute)
   2d. 새 preflight UI 표시

3. 새 preflight에서 [y] → Stage 1 정상 진행
4. 새 preflight에서 [n] → 원래 모델로 복귀 후 abort
```

**왜 중간 stage 전환 불가**:
- Strategy 는 Stage 2a/2c/2d 각각 다른 prompt 와 parse 로직 사용 → 혼합 시 artifact 불일치
- Cache key 에 strategy 포함 → mid-run 변경 시 캐시 의미 상실
- 트랜잭션 무결성 (Stage 1-7 같은 strategy) 보장 필수

**구현 제약**:
- `PREFLIGHT_REENTRY_MAX = 2` — 무한 전환 방지
- Privacy-violated plasmid 있을 때 [c] 선택 → 사용자에게 "{n} plasmids marked privacy: local-only block cloud switch" 명시 거부

**UX (§6 Stage 0 preflight 보강)**:
```
[0/7] Preflight
  Model: llama3.1:8b (local, 8B, local-small tier)
  Expected: ~10min, rollback rate ~15%
  Strategy: field-by-field / extractive / deterministic-only
  
  [y] Continue
  [c] Switch to cloud (restart preflight)
  [n] Abort

> c

Pre-flight for cloud switch:
  ✓ No plasmids marked 'privacy: local-only' (cascade-safe)
  ⚠ Cascade will be enabled for this session (in-memory, config.json unchanged)

  Switch runtime model?
    From: llama3.1:8b
    To:   claude-haiku-4-5 (suggested by cascade config, editable)
  [y/n/e (edit)]
> y

Re-running [0/7] Preflight...
  Model: claude-haiku-4-5 (cloud)
  Strategy: single-pass / abstractive / llm-only
  Expected: ~2min, rollback rate ~5%
  
  [y] Continue  [n] Abort
```

**Rationale**:
- 깨끗한 상태 전이 — strategy 바뀌면 stage 0부터 재평가
- Privacy enforcement (P-1.21 §4) 와 통합 — local-only plasmid는 절대 cloud로 넘어가지 않음
- In-memory override 만 → user config.json 오염 방지 (의도 없는 cascade 영구 활성 예방)

**영향**:
- `src/commands/recombination.ts` preflight re-run 로직 (`reentryCount` 상태)
- `src/recombination/preflight.ts` privacy assertion 호출
- Telemetry `recombination.preflight_rerun{reason}` counter (reason: switch-to-cloud / switch-to-local / model-drift)
