# P-1.6 — 3-Mode Share Pattern 설계서

**작성일**: 2026-04-23
**상태**: **v0.2 — Open Questions 해결 + Exit Criteria 충족** (Phase -1 설계, carryover)
**관련**: PRD §7.1 디렉토리 레이아웃 / archive/hardening §8.3
**구현 시점**: Phase 2 Week 4 (Persistence, Stage 4)

**이 문서의 역할**: 생성된 artifact의 git commit 범위 (`shared` / `local` / `ephemeral`) 를 config에서 명시 + Stage 4 persistence에서 강제.

---

## 0. TL;DR

- 3 mode: **shared** (git 포함), **local** (gitignore), **ephemeral** (recombination 마다 덮어쓰기, gitignore)
- `.dhelix/config.json` 의 `share.default` + plasmid-level override
- `.gitignore` 자동 관리: `.dhelix/plasmids/archive/`, `.dhelix/recombination/`, local/ephemeral artifact 경로
- Share mode 변경 시 cure로 이전 mode artifact 정리 + 경고

---

## 1. 3 Mode 정의

| Mode | git 포함? | 목적 | 예시 artifact |
|-----|---------|-----|-------------|
| **shared** | ✓ | 팀 표준 (모든 팀원이 공유) | PR 리뷰 agent, 프로젝트 OWASP hook, DDD rule |
| **local** | ✗ (.gitignore) | 개인 preference | 개인 코딩 스타일 rule, 본인 언어(한글) 선호 plasmid |
| **ephemeral** | ✗ + recombination 마다 덮어씀 | 실험, 일회성 | "이번 sprint에만 쓸 debug skill" |

// src/config/schema.ts 확장
export const shareConfigSchema = z.object({
  default: z.enum(["shared", "local", "ephemeral"]).default("shared"),
  overrides: z.record(z.string(), z.enum(["shared", "local", "ephemeral"])).default({}),
  // overrides: { "<plasmid-name>": "local" }

// dhelixConfigSchema에 추가

```

    "default": "shared",
    "overrides": {

      "sprint-42-debug": "ephemeral"
    }
  }
}
```

## 4. Artifact 경로 매핑

Stage 4 Persistence에서 artifact 저장 경로 결정:

| Plasmid share | Artifact path | .gitignore 필요? |
|--------------|--------------|----------------|
| shared | `.dhelix/hooks/<name>.ts` 등 기본 경로 | X |
| local | 동일 경로 + `.dhelix/.local-artifacts.json` 에 이름 등록 | .gitignore에 해당 파일들 등록 |
| ephemeral | `.dhelix/ephemeral/<name>.<ext>` | `.dhelix/ephemeral/` 통째로 gitignore |

### 4.1 `.local-artifacts.json`

```json
{
  "hooks": ["my-korean-style-hook.ts"],
  "rules": ["my-korean-style-rule.md"]
}
```

이 파일 자체는 gitignore 대상. local artifact 목록 추적용.

### 4.2 `.gitignore` 자동 관리

`src/plasmids/gitignore-manager.ts` (신규) 가 Stage 7 Release에서:

```typescript
export async function updateGitignore(shareMap: Map<Plasmid, ShareMode>): Promise<void> {
  const lines = new Set<string>();
  // 기본 항목
  lines.add(".dhelix/recombination/");
  lines.add(".dhelix/plasmids/archive/");
  lines.add(".dhelix/ephemeral/");
  lines.add(".dhelix/.local-artifacts.json");

  // local artifact 경로
  for (const [p, mode] of shareMap) {
    if (mode !== "local") continue;
    for (const artifact of getGeneratedArtifactsOf(p)) {
      lines.add(artifact.path);
    }
  }

  await mergeGitignore(".gitignore", lines, "# === dhelix auto-managed ===");
}
```

**Idempotent**: 기존 `.gitignore`에 marker 섹션 (`# === dhelix auto-managed ===`) 안쪽만 관리. 바깥은 user-authored.

## 5. Mode 전환

Plasmid의 share mode가 변경될 때:

### 5.1 shared → local

- 기존 shared artifact를 **git 이력에서 제거 필요** (사용자 수동 권장)
- `.gitignore`에 추가
- `.local-artifacts.json`에 등록

UX:
```
⚠ Changing 'my-korean-style' from shared → local
  The following files will be added to .gitignore:
    - .dhelix/hooks/my-korean-style-hook.ts
    - .dhelix/rules/my-korean-style-rule.md

  ⚠ These files exist in git history. To fully remove them:
    $ git rm --cached .dhelix/hooks/my-korean-style-hook.ts
    $ git rm --cached .dhelix/rules/my-korean-style-rule.md
    $ git commit -m "chore: make my-korean-style local"

Continue? [y/n]
```

### 5.2 local → shared

- `.gitignore`에서 제거
- Warning: "파일이 git에 추가되면 팀원 모두 볼 수 있음"
- `.local-artifacts.json`에서 제거

### 5.3 → ephemeral

- 기존 artifact 삭제 → `.dhelix/ephemeral/`로 이동
- 다음 recombination마다 덮어씀

## 6. Ephemeral Lifecycle

`.dhelix/ephemeral/<name>.ts` 는 매 recombination에서 **우선 삭제 후 재생성**:

```typescript
// Stage 4 Persistence
if (mode === "ephemeral") {
  await rmdir(`.dhelix/ephemeral/${plasmid.name}/`, { recursive: true });
  await mkdir(`.dhelix/ephemeral/${plasmid.name}/`);
  // 그 후 artifact 쓰기
}
```

**Use case**: 실험적 plasmid, sprint-specific debug. Recombination 취소 또는 plasmid 삭제 시 clean.

## 9. Resolved Questions (v0.2)

### Q1 — `git filter-repo` 자동 제공 여부

**확정**: **v0.4 자동 제공 금지. 수동 안내만**.

`/cure` 또는 `shared → local` 전환 시 UX에 아래 안내 출력:
```
⚠ Files removed from tracking, but git history still contains them.
  To fully remove from history (optional, destructive):
    $ git filter-repo --invert-paths --paths <file1> <file2>
  Note: git history rewrite affects all collaborators.
  See docs/team-templates.md for team coordination.
```

**Rationale**: `filter-repo` 는 history 변경 → 팀 협업 파괴 risk. 사용자가 의식적으로 실행해야 안전.

### Q2 — Local artifact hash 검증 (cure 무결성)

**확정**: **`.dhelix/.local-artifacts.json` 에 `{ path, hash, createdAt }` 추가 (v0.4 포함)**. Cure 시 hash mismatch 면 "local artifact was manually edited — reclaim or backup?" 안내.

**Rationale**: Shared artifact는 git 이력으로 이미 hash 추적됨. Local 은 별도 manifest 필요 — v0.4에 기초 레벨 포함 (수동 편집 감지). Phase 3+ 에서 cure 3-way merge 와 통합.

### Q3 — Ephemeral 장수 plasmid 자동 archive

**확정**: **30일 미사용 → `/plasmid list` 실행 시 archive 제안**.

```
$ /plasmid list
...
⚠ Stale ephemeral plasmids (>30 days inactive):
  - sprint-42-debug (last used 2026-03-20, 34 days ago)
  
Archive? [y/n/ignore-session]
```

**Rationale**: Ephemeral ≠ 장수 원칙 구현. 사용자 명시 확인 후 archive (auto-delete 는 아님).

### Q4 — Mono-repo 서브프로젝트별 share 설정

**확정**: **v0.4 각 `.dhelix/` 독립**. Mono-repo 인식 로직 없음.

**Workaround**: 각 서브프로젝트에 별도 `.dhelix/` 디렉토리 + 자체 plasmid. Root-level plasmid 공유는 불가.

**Phase 2+ 확장**: `.dhelix/config.json` `monorepo: { rootConfig: "../.dhelix/config.json", shareScope: "inherit" }` 고려.

**Rationale**: 복잡도 vs. 실제 user base (대부분 single-repo). Phase 0 POC 참가자 중 mono-repo 요청 시 우선순위 조정.
