# P-1.13 — Compression Pipeline 4-Layer 설계서

**작성일**: 2026-04-23
**상태**: **v0.2 — Open Questions 해결 + Exit Criteria 충족** (Phase -1 설계, P-1.14 선행 완료)
**관련**: PRD §6.3.2 Stage 2c / archive/hardening §6.3 / 기존 `src/core/system-prompt-builder.ts`
**구현 시점**: Phase 2 Week 3 (Recombination MVP)

**이 문서의 역할**: "10 plasmid → 1500 tokens"가 어떻게 달성되는지 4-layer 구조를 코드 수준으로 확정한다. Tier-aware ratio, overflow 정책, token budget 재분배 포함.

---

## 0. TL;DR

- **4 Layer**: A (Frontmatter 추출, LLM 없음) → B (LLM abstractive summary) → C (Type bucket 분류, 결정적) → D (Project profile compression)
- **Tier-aware ratio**: foundational 1:1, policy 5:1, tactical 10:1, agent-proposed 15:1
- **Token budget 재설계**: `plasmidSectionsBudget: 1500` (10 plasmid × 150 tokens)
- **Overflow 정책**: priority 낮은 plasmid의 prompt section 드롭, artifact는 유지
- **캐시**: content-addressed (plasmid.hash + layer.version + model.id)
- **파일 출력**: `.dhelix/prompt-sections/generated/` 하위 4개 bucket 파일

---

## 1. 구조 개요

```
Input: [plasmid_1, ..., plasmid_n] + ProjectMeta

Layer A (Frontmatter Extraction)     ← 결정적, LLM 없음
  → Map<name, CompressedLine {id, line, tokens=15}>

Layer B (Abstractive Summary)         ← LLM 1회 per plasmid (cache hit 시 skip)
  → Map<name, AbstractiveSummary {tokens, preservedConstraints[]}>

Layer C (Structural Categorization)   ← 결정적
  → Map<bucket, Summary[]>   // by plasmid.types[0]

Layer D (Project Root Knowledge)      ← LLM 1회 per recombination (cache hit 시 skip)
  → ProjectProfile {text, tokens=300-500}

Output: generated/ 4 파일 (~1500 + 400 tokens)
```

## 2. Layer A — Frontmatter Extraction

**책임**: LLM 없이 plasmid의 metadata만으로 1-line 식별자 생성.

```typescript
// src/recombination/compression/frontmatter-extractor.ts
export interface CompressedLine {
  readonly plasmidName: string;
  readonly line: string;      // `"<name>: <description>"` 형식
  readonly tokens: number;    // 항상 ~15 추정
}

export function extractFromFrontmatter(p: Plasmid): CompressedLine {
  const line = `${p.name}: ${p.description}`.slice(0, 200);
  return {
    plasmidName: p.name,
    line,
    tokens: estimateTokens(line, p.tokenizer),
  };
}
```

**언제 Layer A만으로 충분한가**:
- `description` 자체가 "한 줄 의도"로 충분히 설명적 (PRD §6.1.2 10-500자 강제)
- Plasmid가 artifact-driven (hook/skill이 주 발현 경로) — prompt에 추가 hint 불필요

이 경우 Layer B는 skip하고 Layer C에서 Layer A 출력을 바로 bucket에 배치.

## 3. Layer B — LLM Abstractive Summary

### 3.1 목표

각 plasmid의 body를 `targetTokens` 이하의 **agent-oriented hint**로 요약. artifact와 중복 최소화.

### 3.2 API

```typescript
// src/recombination/compression/plasmid-summarizer.ts
export interface CompressionRequest {
  readonly plasmid: Plasmid;
  readonly targetTokens: number;        // tier 기반 기본값 + user override
  readonly purpose: "identity" | "capability" | "constraint";
  readonly artifactList: readonly string[];  // 이미 생성된 artifact 이름 (중복 회피)
  readonly language: "en" | "ko" | "ja-zh";  // P-1.8 I18n 결과
}

export interface AbstractiveSummary {
  readonly plasmidName: string;
  readonly summary: string;
  readonly tokens: number;
  readonly preservedConstraints: readonly string[];  // "NOT/NEVER" 문구 추적
  readonly cacheHit: boolean;
}

export async function summarizeByLLM(
  req: CompressionRequest,
  llm: LLMClient,
): Promise<AbstractiveSummary>;
```

### 3.3 LLM 프롬프트 구조

```
System: You compress plasmid intents for an AI agent's system prompt.
Rules:
 - Output ${targetTokens} tokens or fewer.
 - For "capability": state WHAT exists and HOW to invoke (skill/hook name).
 - For "constraint": state WHAT NOT to do, imperative voice, preserve NOT/NEVER.
 - For "identity": state HOW the agent should think about this domain.
 - Do NOT duplicate content that appears in generated artifacts (listed below).
 - Do NOT include examples, evidence, or URLs.
 - Preserve user-facing strings in ${language}. All schema fields in English.

User:
Plasmid:
---
${plasmid.body}
---

Generated artifacts (DO NOT re-describe):
${artifactList.join("\n")}

Output JSON (no prose): { summary: string, tokenCount: number, preservedConstraints: string[] }
```

### 3.4 `purpose` 결정 규칙

`plasmid.types` 첫 값으로 매핑:

| types[0] | purpose |
|---------|---------|
| value | identity |
| epistemic | identity |
| structural | constraint |
| ritualistic | capability |
| behavioral | capability |

복합 types의 경우 우선순위: value > structural > epistemic > behavioral > ritualistic.

### 3.5 Cache Key

```typescript
const key = sha256([
  plasmid.hash,              // plasmid .md content hash
  "summarizer",
  SUMMARIZER_PROMPT_VERSION, // semver, prompt 변경 시 bump
  req.targetTokens,
  req.purpose,
  req.language,
  llmModelId,
  llmModelDigest ?? "cloud",
].join("\x00"));
```

캐시 위치: `.dhelix/recombination/objects/<hash>.summary.json` (DD-2 object store 재사용).

### 3.6 Failure Modes

| 실패 | 대응 |
|-----|-----|
| LLM JSON parse fail | 1회 retry. 실패 시 Layer A로 fallback (description만 사용) |
| token count > target | 문자열 끝에서 토큰 단위 truncate + `…` 추가 |
| `preservedConstraints` 누락 | warning + summary 그대로 사용 |
| 로컬 LLM "extractive mode" (P-1.19) | Layer B skip, body의 첫 문단을 truncate로 사용 |

|-----------------|-----------|----------------|---------|

| `epistemic` | `65-domain-knowledge.md` | `## Domain Knowledge` | 75 |

| `ritualistic` | `75-active-capabilities.md` | `## Active Capabilities` | 60 |

### 4.2 Bucket 파일 포맷

각 bucket 파일:
```markdown
<!-- GENERATED — do not edit directly. Sources: owasp-gate, auth-hardening -->

## Active Capabilities

<!-- BEGIN plasmid: owasp-gate -->
Security review orientation: screen for OWASP Top 10 (A01/A03/A07 priority).
Invoke `/security-scan` for explicit scans. PreToolUse hook runs on git commit.
<!-- END plasmid: owasp-gate -->

<!-- BEGIN plasmid: auth-hardening -->
Authentication code requires MFA consideration.
Invoke `@auth-reviewer` for deep review.
<!-- END plasmid: auth-hardening -->
```

BEGIN/END 마커가 cure의 선택적 제거 단위. I-9 준수 (user 영역 건드림 금지 — 이 파일은 100% plasmid-derived).

### 4.3 Assembler API

```typescript
// src/recombination/compression/section-assembler.ts
export function assembleBucket(
  bucketId: BucketId,
  summaries: readonly AbstractiveSummary[],
): BucketFile {
  return {
    path: `.dhelix/prompt-sections/generated/${BUCKET_FILENAMES[bucketId]}`,
    content: [
      `<!-- GENERATED — do not edit directly. Sources: ${summaries.map(s => s.plasmidName).join(", ")} -->`,
      "",
      `## ${SECTION_HEADINGS[bucketId]}`,
      "",
      ...summaries.map(s => [
        `<!-- BEGIN plasmid: ${s.plasmidName} -->`,
        s.summary,
        `<!-- END plasmid: ${s.plasmidName} -->`,
        "",
      ]).flat(),
    ].join("\n"),
    totalTokens: summaries.reduce((n, s) => n + s.tokens, 0),
    plasmidsIncluded: summaries.map(s => s.plasmidName),
  };
}
```

## 6. Tier-Aware Compression Ratio

### 6.1 Plasmid tier → target tokens 매핑

| `tier` | Target tokens | Ratio (vs body 500 tokens) | 비고 |
|-------|--------------|---------------------------|-----|
| `foundational` | 250 | 2:1 | 정보 보존 우선 |
| `policy` | 100 | 5:1 | 기본값 |
| `tactical` | 50 | 10:1 | 공격적 |
| `agent-proposed` | 33 | 15:1 | 최저 confidence |

### 6.2 사용자 override

```yaml
---
name: owasp-gate
prompt-compression:
  level: standard             # minimal | standard | aggressive | none
  budget: 150                 # optional, 강제 상한
  preserve: [constraints]
---
```

| `level` | Ratio multiplier |
|--------|----------------|
| minimal | × 1.5 (덜 압축) |
| standard | × 1.0 (tier 기본값) |
| aggressive | × 0.5 (더 압축) |
| none | 0 tokens (prompt에 안 넣음, artifact만 발현) |

`budget`은 최종 상한. `tier * multiplier < budget`이면 budget이 우선.

`preserve: [constraints]` → LLM 프롬프트에서 "constraints 섹션의 NOT/NEVER 문구는 반드시 포함" 지시 추가.

## 8. Overflow Policy

`plasmidSectionsBudget` 초과 시:

```typescript
export interface OverflowDecision {
  readonly keeps: readonly AbstractiveSummary[];
  readonly drops: readonly { summary: AbstractiveSummary; reason: string }[];
  readonly userConfirmationRequired: boolean;
}

export function resolveOverflow(
  summaries: readonly AbstractiveSummary[],
  budget: number,
): OverflowDecision {
  const sorted = [...summaries].sort(comparePlasmids);  // 높은 priority 먼저
  let remaining = budget;
  const keeps: AbstractiveSummary[] = [];
  const drops: OverflowDecision["drops"] = [];

  for (const s of sorted) {
    if (remaining - s.tokens >= 0) {
      keeps.push(s);
      remaining -= s.tokens;
    } else {
      drops.push({ summary: s, reason: `over budget (${s.tokens} tokens, remaining ${remaining})` });
    }
  }
  return { keeps, drops, userConfirmationRequired: drops.length > 0 };
}
```

### 8.1 Priority 비교 규칙

```typescript
function comparePlasmids(a: Plasmid, b: Plasmid): number {
  // 1. tier (foundational > policy > tactical > agent-proposed)
  if (tierOrder(a.tier) !== tierOrder(b.tier)) return tierOrder(a.tier) - tierOrder(b.tier);
  // 2. priority (high > normal > low)
  if (prioOrder(a.priority) !== prioOrder(b.priority)) return prioOrder(a.priority) - prioOrder(b.priority);
  // 3. 이름 알파벳 (결정성)
  return a.name.localeCompare(b.name);
}
```

### 8.2 UX — Stage 3 Preview

```
⚠ Token budget exceeded for plasmid sections
  Budget:      1,500 tokens
  Requested:   2,340 tokens
  Overflow:    840 tokens

Drops (lowest priority first):
  - agent-proposed:react-hooks-order (confidence 0.78)  — 180 tokens
  - tactical:verbose-logging                            — 220 tokens

Keeps (higher priority):
  - foundational:core-values                            — 320 tokens
  - policy:owasp-gate                                   — 180 tokens
  [... 7 more]
```

**핵심 원칙**: drop된 plasmid도 **artifact는 유지** (hook, skill은 계속 동작). prompt-level hint만 생략되어 agent의 자발적 참조가 약해질 뿐.

## 11. Resolved Questions (v0.2)

### Q1 — `plasmidSectionsBudget` 적정값

**결정**: **Adaptive budget 채택** — 하드코딩 1500 대신 실제 활성 plasmid 수 기반 동적 계산.

**공식 (§7.2 대체)**:

```typescript
// src/core/system-prompt-builder.ts
export const PLASMID_TOKEN_TARGET = 150;   // per plasmid
export const PLASMID_BUDGET_FLOOR = 300;   // 2 plasmid 분량 (솔로 사용자 보호)
export const PLASMID_BUDGET_CEIL  = 1500;  // 10+ plasmid에서 포화

export function resolvePlasmidBudget(
  activeCount: number,
  capabilityTier: CapabilityTier,
): number {
  const raw = activeCount * PLASMID_TOKEN_TARGET;
  const clamped = Math.max(PLASMID_BUDGET_FLOOR, Math.min(PLASMID_BUDGET_CEIL, raw));
  // local 환경 비례 축소 (P-1.20 §2.5 budget row와 일치)
  return capabilityTier === "low"    ? Math.round(clamped * 0.33)
       : capabilityTier === "medium" ? Math.round(clamped * 0.66)
       : clamped;
}
```

**Rationale**:
- 경쟁 도구 실측: Cursor `.cursorrules` 평균 ≈3K tokens, Claude Code CLAUDE.md 2-10K. 1500 상한은 합리적
- 솔로 사용자 (3-5 plasmid) 에게 1500 하드코딩은 budget **낭비** — 다른 context 확보 차단
- `Math.max(floor, ...)` 로 초기 사용자 (2-3 plasmid) 도 충분한 공간 확보
- Capability tier 연동으로 local-small에서 500 tokens 자동 축소 (P-1.20 §2.5와 일치)
- Phase 0 POC 이후 `PLASMID_TOKEN_TARGET` 상수만 변경 가능 (PR 1개)

**영향**:
- §7.2 `PromptBudget.plasmidSectionsBudget: number` → **함수 호출 기반으로 대체**
- `src/core/system-prompt-builder.ts` Phase 2 W3 구현 시 `resolvePlasmidBudget()` 경유
- Telemetry `plasmid.resolved_budget{tier}` gauge 추가 (실사용 중앙값 추적 → 향후 tuning 근거)

### Q2 — 복합 `types` 의 bucket 배치

**결정**: **첫 값 주 배치 + 명시적 opt-in 복제** (`secondary-bucket` 필드 추가).

**Schema 확장**:
```yaml
types: [value, epistemic]
secondary-bucket: true     # optional, default false — 두 번째 bucket에도 복제 주입
```

효과 (`secondary-bucket: true` 시):
- `60-principles.md` (value, primary) + `65-domain-knowledge.md` (epistemic, secondary) 두 파일에 BEGIN/END 쌍 동일 summary 주입
- Token 사용량 2배 → overflow governor가 drop 결정에 반영

**Rationale**:
- "첫 값만" 의 정보 손실: `[value, epistemic]` plasmid가 principles bucket만 있으면 epistemic 맥락 요청 시 누락
- 자동 복제는 토큰 2배 + overflow 위험 — **사용자 판단 영역** (opt-in 철학)
- 기본값 OFF → 대부분의 plasmid는 첫 값만 (단순, 결정적)

**영향**:
- `src/plasmids/schema/v1.ts` — `secondary-bucket: z.boolean().default(false)` 필드
- `src/recombination/compression/section-assembler.ts` — `secondary-bucket=true` 면 `types[1]` bucket에도 summary 주입
- Overflow governor (§8) — 복제된 토큰도 total 계산 포함

### Q3 — `level: none` artifact 전용 모드 확정

**결정**: **Prompt 주입 없음, artifact는 registry discover에만 의존**.

**Flow (§3, §4 보강)**:
- `level: none` plasmid → Layer A 실행 (description만 추출) → **bucket 파일에 쓰지 않음**
- Layer B skip, Layer C assembler 가 제외
- Artifact (hook, skill, command) 는 여전히 생성되어 file system에 존재 → runtime registry scan 으로 자동 discover

**Rationale**:
- Plasmid-level prompt hint는 agent의 **자발적 참조 빈도** 영향. artifact 자체 동작은 독립
- Hook: event trigger 시 강제 발동 (prompt 무관)
- Skill: `/command` 명시 호출 또는 registry scan 으로 discover
- Command: `/help` 등에 자동 등재
- 따라서 `level: none` 은 "artifact만 있으면 됨" 케이스 (예: 단순 pre-commit 체크) 에 최적

**영향**:
- Overflow governor에서 `level: none` 은 계산 대상 0 토큰
- Stage 3 Preview: "`N plasmids with level=none: prompt-silent artifacts only`" 표시
- Telemetry `compression.level_none_count` counter

### Q4 — Compression 품질 측정 (2-Stage)

**결정**: **결정적 Stage 1 (Phase 2 W4) + LLM judge Stage 2 (Phase 3, optional weekly audit)**.

**Stage 1 — 결정적 지표 (`src/recombination/compression/quality.ts` 신규)**:
```typescript
export interface DeterministicQuality {
  /** preservedConstraints 매칭 비율 — Layer B 프롬프트가 NOT/NEVER 문구를 유지했는가 */
  readonly preservedConstraintsRatio: number;    // 0.0-1.0
  /** summary_tokens / target_tokens — 1.0 이상이면 overflow, <0.7 이면 under-utilization */
  readonly tokenEfficiency: number;
  /** summary 내 plasmid body section 참조 개수 / body 총 section 수 */
  readonly bodyCoverageRatio: number;
}
```
매 recombination에서 자동 계산. Metric `compression.deterministic_quality{plasmid, layer, metric}` histogram.

**Stage 2 — LLM judge (Phase 3, optional)**:
```
Prompt (cloud only, 로컬 시 skip):
"Rate how well this summary preserves the original plasmid's semantic intent.
 1-5 scale on: (1) factual fidelity, (2) constraint preservation, (3) actionability.
 Output JSON: { score: number, breakdown: {...}, flags: string[] }"
```
- `.dhelix/config.json` `compression.qualityAudit: "never" | "weekly" | "per-recomb"` (default `"never"`)
- Metric `compression.semantic_quality` histogram

**Rationale**:
- Stage 1 단독은 "문구 유지는 했지만 의미 뒤집힘" 케이스 못 잡음
- Stage 2 LLM judge 가 semantic drift 주 방어선이지만 비쌈 + 로컬 불가
- → 결정적이 **primary** (매 build), LLM은 **audit** (weekly/on-demand)

**영향**:
- `src/recombination/compression/quality.ts` 신규 (~80 LOC, Phase 2 W4)
- Phase 3에서 `src/recombination/compression/quality-llm-judge.ts` 추가
- `.dhelix/config.json` `compression.qualityAudit` 필드
