## 2. Quick Mode Flow (기본, 3-step)

### 2.1 Step 1 — Intent Parsing (3s)

```
[사용자] /plasmid "OWASP 검사 강제"

[dhelix] [1/3] Intent parsing (3s)
              name: owasp-gate          (auto-generated from intent)
              scope: [hooks, rules]     (inferred)
              types: [structural]       (inferred)
              priority: normal          (default)
              tier: policy              (default)
```

**Inference 알고리즘** (결정적):
- `name`: intent를 한글 → romanize → kebab-case → 15자 이하
- `scope`: intent 키워드 매핑
  - "검사", "체크", "enforce" → `hooks, rules`
  - "리뷰", "review" → `agents`
  - "도우미", "assistant" → `skills`
  - "커맨드", "command" → `commands`
- `types`: 위와 유사
  - "강제", "금지" → `structural`
  - "스타일", "매너" → `behavioral`
  - "지식", "정보" → `epistemic`
  - "원칙", "가치" → `value`
- `priority`: always `normal`
- `tier`: always `policy`

이 모든 추론은 **사용자 수정 가능**. Step 3에서 `[e]` 플래그로 에디터 열기.

### 2.2 Step 2 — Draft Generation (15s)

```
         [2/3] Draft generation (15s)
              ┌──────────────────────────────────────┐
              │ .dhelix/plasmids/owasp-gate.md       │
              │                                      │
              │ ---                                  │
              │ name: owasp-gate                     │
              │ description: "OWASP Top 10 강제"     │
              │ scope: [hooks, rules]                │
              │ types: [structural]                  │
              │ priority: normal                     │
              │ tier: policy                         │
              │ active: true                         │
              │ ---                                  │
              │                                      │
              │ ## Intent                            │
              │ 모든 커밋 전에 OWASP Top 10 검사.    │
              │                                      │
              │ ## Behavior                          │
              │ (LLM-generated draft)                │
              │                                      │
              │ ## Constraints                       │
              │ (LLM-generated draft)                │
              └──────────────────────────────────────┘
```

**LLM 프롬프트** (단 한 번 호출):
```
Generate a minimal plasmid body for this intent: "${userIntent}"
Frontmatter (already filled): ${JSON.stringify(inferredFrontmatter)}

Required sections:
- ## Intent (1-2 sentences, ${language})
- ## Behavior (specific actions, ${language})
- ## Constraints (2-3 bullets, what NOT to do, ${language})

Optional: ## Evidence (if user mentioned a standard/reference)

Output: markdown body (no frontmatter, no ```).
```

**예산**:
- Cloud: ~15s (1 LLM call)
- Local-large: ~30s
- Local-small: ~60s + retry

### 2.3 Step 3 — 저장 옵션

```
         [3/3] 저장
              [y] 저장 + /recombination 자동 실행
              [s] 저장만 (recombination 수동)
              [e] 에디터 편집 후 저장 ($EDITOR)
              [r] Research 모드로 심화 (--research flag)
              [c] 취소
```

**`[y]` 선택**: plasmid .md 저장 + `/recombination` 자동 발동 (사용자 명시 승인 취급).
**`[e]` 선택**: `$EDITOR` 환경변수 우선, 없으면 `vi`/`nano`. 에디터 종료 후 저장.
**`[r]` 선택**: 현재 draft를 research flow 입력으로 넘김 (draft 유지, §3.3 research 진입).

## 10. Resolved Questions (v0.2)

### Q1 — Intent inference 언어 지원 범위

**확정**: **v0.4 = 한글 + 영어. 일본어/중국어 = Phase 2**.

- 한글: Hangul Syllables 10% 이상 → 한글 키워드 룰 적용 (`검사`, `리뷰`, `강제` 등)
- 영어: ASCII dominant → 영어 키워드 룰 (`check`, `enforce`, `review`)
- 기타 언어 (일본어/중국어): fallback 모드 — inference 없이 빈 frontmatter + "edit manually" 안내

**Rationale**: P-1.8 I18n과 일관. Phase 0 POC 참가자 대부분 한/영이므로 우선순위 맞음.

**영향**: `src/plasmids/intent-inference.ts` 에 `supportedLanguages: ["ko", "en"]` 상수 + unsupported 시 fallback.

### Q2 — Research mode 전환 context 전달

**확정**: **intent 문자열 + inferred frontmatter 모두 context 전달**. Research가 frontmatter 변경 가능 (예: scope 변경, tier 추가).

**구현**:
```typescript
interface ResearchContext {
  originalIntent: string;
  currentDraft: Partial<Plasmid>;    // Quick mode 결과
  allowFrontmatterEdit: true;
}
```

**Rationale**: Research가 "심화" 모드인만큼 Quick 결과의 모든 context 활용 가능해야. Frontmatter 잠그면 research의 가치 반감.

### Q3 — `--dry-run` 출력 캐싱

**확정**: **`.dhelix/plasmids/.drafts/<name>.md` 임시 저장**. gitignore 자동 포함. 7일 후 auto-cleanup.

```
/plasmid list --drafts       # draft 목록
/plasmid save-draft <name>   # draft → active
/plasmid rm-draft <name>     # 수동 삭제
```

**Rationale**: 재실행 비용 감소 (Quick mode 20s + LLM cost). 임시 저장 경로 명확하게 분리.

### Q4 — Quick mode 의 `tier: foundational` 자동 inference

**확정**: **금지**. Foundational tier 선언은 **명시적 opt-in only**.

경로:
1. `/plasmid --tier foundational "..."` 플래그
2. `/plasmid --template foundational-core` 템플릿
3. 에디터에서 `tier: foundational` 수동 입력 + `challengeable-by` 필드

**Rationale**:
- Foundational 은 challenge cost 높은 언약 — 자동 inference 는 사용자가 의도 모르고 foundational 생성 위험
- P-1.10 `challengeable-by` 의 `min-justification-length` 같은 세부 규칙 설정 필요 — Quick mode 에서 자동 불가
