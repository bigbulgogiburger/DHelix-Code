# DB Inc. PPTX Design System Reference

이 문서는 DB Inc. 공식 프레젠테이션의 디자인 시스템을 정의합니다.
`scripts/build_pptx.py`가 이 규칙을 자동으로 적용합니다.

## 슬라이드 규격

- **크기**: 13.33 × 7.50 인치 (16:9 와이드스크린)
- **단위**: EMU (914400 EMU = 1 inch)
- Width: 12,192,000 EMU
- Height: 6,858,000 EMU

## 폰트 체계

DB Inc.는 자체 폰트 패밀리 3종을 사용합니다:

| 폰트명                | 용도                                   | 크기 범위   |
| --------------------- | -------------------------------------- | ----------- |
| **DB제목 B** (Bold)   | 섹션 헤더 바, 페이즈 타이틀, 핵심 라벨 | 12pt ~ 16pt |
| **DB제목 M** (Medium) | 테이블 헤더, 서브 라벨, 중간 강조      | 10pt ~ 13pt |
| **DB제목 L** (Light)  | 테이블 본문, 설명 텍스트               | 10pt ~ 14pt |

### 폰트 적용 규칙

- **표지 슬라이드**: 제목/부제목은 마스터 레이아웃 상속 (DB제목 B 계열)
- **슬라이드 번호** (01, 02...): 상속 + 색상 `#002060`
- **페이지 제목**: 상속 (마스터 레이아웃)
- **요약 텍스트**: 14pt Bold (DB제목 계열 상속)
- **섹션 헤더 바**: DB제목 B, 12pt, 중앙정렬
- **테이블 헤더행**: DB제목 M, 12pt, Bold
- **테이블 라벨 열**: DB제목 M, 12pt, Bold
- **테이블 본문**: DB제목 L, 10~11pt
- **Phase 타이틀**: DB제목 B, 16pt, 중앙정렬
- **Phase 설명**: DB제목 M, 13pt, 중앙정렬
- **기대효과 라벨**: DB제목 M, 10pt, Bold, 중앙정렬
- **기대효과 본문**: DB제목 M, 12pt, Bold, 중앙정렬

## 컴포넌트 패턴

### 섹션 헤더 바 (Section Header Bar)

- **형태**: 둥근 모서리 사각형 (AutoShape, round corner)
- **높이**: 295,275 EMU (~0.32 인치)
- **너비**: 내용 영역 전체 폭 (~10,700,000 EMU)
- **폰트**: DB제목 B, 12pt, 중앙정렬
- **색상**: 테마 색상 (마스터 레이아웃에서 상속)
- **위치**: 테이블 바로 위, 약간의 간격

### 테이블 (Table)

- **헤더행**: DB제목 M, 12pt, Bold
- **라벨 열 (첫 번째 열)**: DB제목 M, 12pt, Bold
- **본문 셀**: DB제목 L, 10~11pt
- **셀 내 줄바꿈**: 파이프(`|`)로 구분하여 줄 구분
- **열 너비**: 라벨 열은 좁게 (700,000~2,200,000 EMU), 본문 열은 넓게

### Phase 카드 (로드맵 슬라이드)

```
┌─── Phase Ⅰ (~3개월) ───┐
│      내부 효율화         │  ← DB제목 B, 16pt
│ ─────────────────────── │
│ 핵심 프로세스 선정...    │  ← DB제목 M, 13pt
│ ┌───────────────────┐  │
│ │ 테이블 (상세)      │  │
│ └───────────────────┘  │
│   ┌── 기대효과 ──┐     │
│   │ 테스트 작성   │     │  ← DB제목 M, 12pt Bold
│   │ 90% 이상 감소 │     │
│   └──────────────┘     │
└────────────────────────┘
```

## 톤 앤 매너

### 문장 스타일

1. **요약 텍스트 (상단)**: 2줄 구성

   - 1줄: `본 <주제>는 <목적>을 위한 것임`
   - 2줄: `<기대효과>를 통해 <가치>를 실현하고자 함`
   - `|` 기호로 줄 구분

2. **테이블 내용**: 간결한 명사형 종결

   - 좋음: `AI 기반 자동 코드 생성`
   - 나쁨: `AI를 기반으로 자동으로 코드를 생성합니다`

3. **섹션 헤더**: 명사구 (동사 없음)

   - 좋음: `PoC 목표`, `실행계획 및 일정`, `평가기준`
   - 나쁨: `PoC 목표를 정의합니다`

4. **수치 강조**: 구체적 숫자 포함
   - `90% 이상 절감`, `평균 10분 이내`, `3개월`

### 슬라이드 구성 패턴

1. **표지**: 프로젝트명 + 부서/회사명 + 날짜
2. **목표 및 핵심과제**: 목표 테이블 + 핵심과제 다이어그램
3. **실행계획**: 범위 테이블 + 일정/마일스톤 + 주차별 계획
4. **프로세스/아키텍처**: 흐름도 또는 단계별 그룹
5. **평가기준**: 정량화 테이블 + 최소 성능 요구사항
6. **로드맵**: Phase 카드 3개 (단기/중기/장기)
7. **EOD**: 마지막 페이지
