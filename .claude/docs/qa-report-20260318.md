## 2. 발견된 버그 (3건)

### BUG-1 [Critical] Ctrl+D 이중 핸들러 충돌

- **파일**: `src/cli/components/UserInput.tsx:238` + `src/cli/App.tsx:340`
- **증상**: 입력 버퍼에 텍스트가 있을 때 Ctrl+D를 누르면, UserInput에서 forward-delete를 수행하지만 동시에 useKeybindings의 `exit` 액션이 `process.exit(0)`을 호출하여 **앱이 즉시 종료**됨
- **원인**: Ink의 `useInput`은 모든 등록된 리스너에 이벤트를 전달하므로, UserInput과 useKeybindings 양쪽이 모두 Ctrl+D를 처리
- **영향**: 사용자가 입력 중 Ctrl+D로 문자를 삭제하려 하면 앱이 종료됨
- **수정 방안**: useKeybindings의 `exit` 핸들러에서 입력 버퍼가 비어 있는지 확인하는 가드 추가, 또는 Ctrl+D를 DEFAULT_BINDINGS에서 제거하고 UserInput에서만 처리

### BUG-2 [Medium] 빈 프롬프트 `-p ""` 대화형 모드 진입

- **파일**: `src/index.ts:468`
- **증상**: `node dist/index.js -p "" --output-format json` 실행 시 프로세스가 무한 대기 (대화형 모드 진입)
- **원인**: `if (opts.print)` 조건에서 빈 문자열 `""`이 falsy로 평가됨
- **영향**: 스크립트에서 빈 프롬프트를 파이프하면 행(hang)
- **수정 방안**: `if (opts.print !== undefined)`로 변경하거나, 빈 문자열 시 에러 메시지 출력 후 종료

### BUG-3 [Medium] Windows에서 Skill 디렉토리 경로 추출 실패

- **파일**: `src/skills/manager.ts:157`
- **증상**: `lastIndexOf("/")` 사용으로 Windows 백슬래시 경로에서 -1 반환 → `skillDir`이 빈 문자열
- **원인**: `path.dirname()` 대신 수동 문자열 처리 사용
- **영향**: Windows에서 `${DHELIX_SKILL_DIR}` 변수가 빈 문자열로 치환
- **수정 방안**: `path.dirname(skill.sourcePath)` 사용
