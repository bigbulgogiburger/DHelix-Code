> 작성일: 2026-03-18

## 채점 기준

### 점수 체계 (항목당 최대 10점)

| 점수 | 기준                                                                |
| ---- | ------------------------------------------------------------------- |
| 10   | 완벽 — 모든 통과 기준 충족, 추가 턴 없이 한 번에 완료               |
| 8-9  | 우수 — 모든 기준 충족, 1-2회 자가 수정                              |
| 6-7  | 양호 — 대부분 충족, 사소한 이슈 1-2건 (빌드/린트 경고 등)           |

| 0-1  | 완전 실패 — 무한 루프, 할루시네이션, 잘못된 파일 수정               |

## 실행 방법론 (QA 에이전트 필독)

> **⚠️ 중요**: QA 에이전트는 테스트 내용을 **직접 구현하지 않습니다**.
> 반드시 `dhelix` CLI를 실제로 실행하고 **dhelix의 응답을 채점**해야 합니다.

### 올바른 실행 흐름

```
QA 에이전트
  └─► dhelix CLI 호출 (멀티턴)
        └─► dhelix가 LLM 호출
              └─► dhelix가 도구 실행 (파일 생성, bash 실행 등)
                    └─► QA 에이전트가 결과 채점
```

### 멀티턴 실행 방법

각 TC는 여러 프롬프트를 **순서대로 dhelix에 입력**합니다.

```bash
# 작업 디렉토리 준비
mkdir -p /tmp/dhelix-e2e/tc01 && cd /tmp/dhelix-e2e/tc01

# 턴 1: 첫 번째 프롬프트를 dhelix에 전달 (headless 모드)
dhelix -p "피보나치 수열을 계산하는 Python 함수를 만들고 실행해줘" \
  --output-style text > turn1.txt 2>&1

# 턴 2: 후속 프롬프트 (이전 컨텍스트 유지 필요 시 --session 플래그 사용)
dhelix -p "fib(10)의 결과를 확인해줘" \
  --output-style text > turn2.txt 2>&1

# 결과 확인
cat turn1.txt turn2.txt
```

### 채점 시 주의사항

| 항목        | 올바른 방법                 | 잘못된 방법               |
| ----------- | --------------------------- | ------------------------- |
| 파일 생성   | dhelix가 생성했는지 확인    | QA 에이전트가 직접 생성   |
| 빌드/테스트 | dhelix가 실행했는지 확인    | QA 에이전트가 직접 실행   |
| 채점        | dhelix 결과물 기준으로 채점 | QA 에이전트 구현물로 채점 |

### dhelix CLI 설치 확인

```bash
# dhelix가 설치되어 있는지 확인
which dhelix || npx dhelix --version

# 또는 소스에서 직접 실행
node /path/to/dhelix/dist/index.js --version
```

## Level 1: Basic (2-3 턴, 단일 도구)

### TC-01: 단일 파일 생성 및 실행

- **프롬프트**: "피보나치 수열을 계산하는 Python 함수를 만들고 `python fibonacci.py 10`으로 실행해서 결과를 보여줘"
- **예상 턴**: 2-3
- **사용 도구**: file_write, bash_exec
- **통과 기준**:
  - [ ] 파일이 생성됨
  - [ ] 함수가 올바른 결과 반환 (fib(10) = 55)
  - [ ] 실행 결과가 출력됨
- **채점**: 파일 생성 3점 + 올바른 로직 4점 + 실행 성공 3점

### TC-02: 파일 읽기 및 정보 추출

- **프롬프트**: "package.json을 읽고 프로젝트 이름, 버전, 의존성 개수를 알려줘"

### TC-03: 간단한 버그 수정

- **프롬프트**: "sum1to10.js를 읽어보고 버그를 찾아서 수정해줘. 1부터 10까지 합은 55여야 해"

  - [ ] 수정 후 실행 결과 55
  - [ ] 최소한의 변경만 적용
- **채점**: 읽기 선행 2점 + 버그 식별 3점 + 수정 정확 3점 + 최소 변경 2점
