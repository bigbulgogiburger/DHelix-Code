### Spring.io 핵심 디자인 패턴 (상세 분석 결과)

**색상 팔레트:**

- 배경: `#ffffff` (화이트)
- 사이드바 active: `#F4F8FA` (쿨 오프화이트) + 좌측 `6px solid #6DB33F` (녹색)
- 보더: `#D9E9E9` (틸 그레이 — 순수 회색이 아닌 약간 청록 틴트)
- 본문: `#111111` (거의 검정), 보조: `#616161` (회색)
- 링크: `#086DC3` (블루)
- 드롭다운 그림자: `rgba(108,135,135,0.5) 0 5px 30px` (틸 틴트 그림자)

**타이포그래피:**

- 헤딩: Metropolis (400~500), H1 40px, H2 22px
- 본문: Open Sans 16px/24px (line-height 1.5)
- 탭: 16px bold, uppercase
- 배지: 12px bold

**레이아웃:**

- 컨테이너: max-width 1344px
- 사이드바: 318px (전체의 22%)
- 카드: border `1px solid #D9E9E9`, 라운드 없음 (sharp corners)
- 그림자: 드롭다운에만 사용, 카드에는 그림자 없음
- 장식: 최소 — 푸터에 큰 녹색 원 하나만

**핵심 패턴:**

| 패턴               | Spring.io 구현                                   | 우리에게 적용                            |
| ------------------ | ------------------------------------------------ | ---------------------------------------- |
| **밝은 배경**      | `#fff` + `#F4F8FA` (사이드바)                    | 라이트 기본 + 다크 토글                  |
| **좌측 사이드바**  | 318px, 녹색 left-border active                   | Layer별 모듈 트리, accent 컬러 active    |
| **콘텐츠 탭**      | OVERVIEW / LEARN / SUPPORT (uppercase, bold)     | 개요 / 레퍼런스 / 사용법 / 내부구현      |
| **버전 배지**      | 컬러 코딩 (green=CURRENT, red=GA, blue=SNAPSHOT) | Layer 배지 (컬러 코딩 유지)              |
| **깔끔한 타이포**  | 2-font (Metropolis + Open Sans)                  | Pretendard 단일 (한글 최적화)            |
| **단일 accent**    | 녹색 `#6DB33F` 하나만                            | 인디고 `#6366f1` 하나 + Layer 색상       |
| **틸 틴트 보더**   | `#D9E9E9` (순수 회색이 아님)                     | `#e2e8f0` (슬레이트 200, 약간 블루 틴트) |

## 3. 리디자인 방향

### 핵심 전환: "화려한 다크 쇼케이스" → "읽기 편한 문서 사이트"

현재 사이트는 랜딩 페이지처럼 디자인되어 있지만, 실제 목적은 **개발 문서**입니다.
문서 사이트의 목적은 "와 멋있다"가 아니라 **"아 이해했다"**입니다.

### 3-1. 테마 전환: 다크 → 라이트 (듀얼 모드)

```
기존 (다크 only):
- bg: #0c1222, text: #f0f4fc
- 코드도 어둡고 배경도 어두워서 구분 안 됨

변경 (라이트 기본 + 다크 토글):
- bg: #ffffff, text: #1a1a2e
- 코드 블록만 다크 배경 유지 → 시각적 구분 확실
- 다크 모드 토글 제공 (개발자 선호 존중)
```

### 3-2. 레이아웃 전환: 단일 스크롤 → 사이드바 + 콘텐츠

```
┌──────────────────────────────────────────────────┐
│  dbcode Guide          Deep Dive  Source Ref  🌙  │
├─────────────┬────────────────────────────────────┤
│             │                                    │
│  LAYER 1    │  📄 agent-loop.ts                  │
│  └ App.tsx  │  Layer 2: Core                     │
│             │  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  LAYER 2    │                                    │
│  ├ agent-loop│  개요  레퍼런스  사용법  내부구현    │
│  ├ context-  │  ──────────────────────────────── │
│  ├ circuit-  │                                    │
│  ├ recovery- │  Agent Loop는 dbcode의 심장입니다.  │
│  └ ...      │  사용자 입력을 받아 LLM을 호출하고,  │
│             │  도구를 실행하고, 결과를 피드백하는   │
│  LAYER 3    │  ReAct 패턴의 메인 실행 루프입니다.  │

│             │  └──────────────────────────────┘ │
│  LAYER 4    │                                    │
│  ├ config    │  ## 레퍼런스                       │
│  └ skills   │  AgentLoopConfig interface...      │
│             │                                    │
└─────────────┴────────────────────────────────────┘
```

### 톤앤매너 원칙

| 원칙                | 설명                                                            |
| ------------------- | --------------------------------------------------------------- |
| **깔끔함 > 화려함** | 그라디언트, 글로우, 글래스 효과 전부 제거. 깨끗한 화이트 + 보더 |
| **가독성 최우선**   | 밝은 배경에 진한 텍스트 (contrast ratio 7:1 이상)               |
| **코드만 다크**     | 코드 블록만 slate-800 배경 → 본문과 확실히 구분                 |
| **단일 accent**     | 인디고(#6366f1) 하나가 사이트 전체 브랜드 컬러                  |
| **장식 최소화**     | noise texture, heroGlow, glass-card 등 제거. Spring처럼 flat    |
| **한글 가독성**     | Pretendard, break-keep-all, line-height 1.6~1.75                |
| **전문적인 느낌**   | 포트폴리오가 아닌 개발 문서. Spring.io, React.dev 수준          |

### 3-4. 타이포그래피 개선

```
기존:
- 헤딩: clamp(2.5rem, ...) font-black — 너무 크고 굵음
- 본문: text-lg — 적절
- 간격: 글래스 카드에 파묻혀서 구분 안 됨

변경:
- 헤딩: text-3xl font-bold (큰 섹션), text-xl font-semibold (서브)
- 본문: text-base, color: gray-700, line-height: 1.75
- 코드: text-sm, font-mono, bg-slate-800 (다크 코드 블록)
- 간격: 섹션 간 py-12, 요소 간 mb-6
```
