# ${name}

Multi-step commit-message review combining subject, body, and footer rules.

## Rules

1. **Subject line** — Conventional Commits: `<type>(<scope>): <subject>`.
   - Imperative mood ("add", not "added"/"adds").
   - No trailing period; max 72 characters including type/scope.
2. **Body** — when present, separated from the subject by one blank line.
   - Wrap at 100 columns.
   - Explain *why*, not *what* the diff already shows.
3. **Footer** — optional, one of:
   - `BREAKING CHANGE: <description>`
   - `Refs: #<issue>` or `Closes: #<issue>`
   - `Co-Authored-By: Name <email>`
4. **Forbidden** — no emojis, no "wip"/"misc"/"stuff" placeholders, no references
   to tooling banners such as "Generated by ...".

## Eval cases

- id: full-commit
  description: a well-formed multi-line commit passes.
  input: |
    feat(plasmids): add template registry

    Introduce a resolver keyed by template id so Quick mode and /plasmid
    create can surface consistent layer metadata across sessions.

    Refs: #GAL-1
  expectations:
    - pattern:^feat\(plasmids\):
    - contains:Refs:

- id: reject-past-tense
  description: a past-tense subject must be flagged.
  input: "fix(loader): fixed orphan detection for two-file lock"
  expectations:
    - not-contains:fixed
